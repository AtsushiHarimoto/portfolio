# 外部 AI 協作指南 (GPT Projects Loop)

## 目的

建立一個標準化的流程，將外部 AI（如 ChatGPT Projects, Claude Projects）的高級架構建議或代碼審查意見，轉化為 Antigravity 可執行的具體任務。

## 反饋收件箱 (Inbox)

所有來自外部 AI 的建議文件，統一存放在：
`workspace/ideas/`

建議命名格式：

- `YYYYMMDD_{Topic}.md`
- 例如：`20260103_ArchReview.md`, `20260104_RefactorPlan.md`

---

## 協作流程

### 1. 導出上下文 (Local -> External)

使用 `/run-sync-docs` 生成全量文檔備份：

- `output/sync_{date}/game_docs.md` (前端)
- `output/sync_{date}/api_test_tool_docs.md` (後端)
- ...

將這些文件上傳至 GPT Projects / Claude Projects。

### 2. 獲取建議 (External -> Local)

在外部 AI 中進行分析討論後，要求其總結建議。

**提示詞 (Prompt)**：

```text
請總結我們剛才的討論，並生成一份給開發代理 (Antigravity) 的執行清單。
請包含：
1. 修改摘要
2. 具體涉及的文件
3. 詳細的修改指令或代碼片段
```

將外部 AI 的回覆全文複製，保存到 `workspace/ideas/YYYYMMDD_{Topic}.md`。
（此目錄已納入 git 版本控制，作為決策記錄的一部分）

### 3. 執行任務 (Local Action)

在 Antigravity 中指令：

> "請查看 workspace/ideas/YYYYMMDD\_{Topic}.md，分析其中的建議，並生成一份 implementation plan。"

Antigravity 將：

1. 讀取反饋文件
2. 提取 Action Items
3. 生成 PLAN
4. 等待用戶批准後執行 (ACTION)

---

## 最佳實踐

- **保持上下文同步**：每次重大變更後，重新運行 `/run-sync-docs` 並更新外部 AI 的知識庫。
- **結構化指令**：如果可能，要求外部 AI 使用 Markdown Checklists 或代碼塊來列出任務，這樣 Antigravity 更容易解析。

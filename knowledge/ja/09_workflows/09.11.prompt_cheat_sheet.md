# AI 協業コマンド早見表 (Cheat Sheet)

> コマンドを忘れましたか？プロジェクトルートで `./help.ps1`（Win）または `./help.sh`（Mac/Linux）を実行すると、インタラクティブメニューが表示されます。

---

## コア開発

| タスク           | コマンド (Prompt) | 説明                                                    |
| ---------------- | ----------------- | ------------------------------------------------------- |
| **新機能開発**   | `/run-dev`        | 標準開発 SOP：ドキュメント確認 -> 計画 -> 実行 -> 検証  |
| **バグ調査**     | `/run-bg`         | 体系的バグ診断：証拠収集 -> 根本原因分析 -> 修正提案    |
| **テスト実行**   | `/run-qa`         | テスト SOP：計画策定 -> テスト実行 -> レポート生成      |

## バージョン管理とアーカイブ

| タスク           | コマンド (Prompt)      | 説明                                                         |
| ---------------- | ---------------------- | ------------------------------------------------------------ |
| **コードコミット** | `/run-archive`       | **推奨**。diff を自動分析し `1.repo.md` 生成、git コミット   |
| **ログ更新**     | `/run-update-blocks`   | 本日のコミットをスキャンし `workspace/work/blocks/` を自動更新 |

## ナレッジ管理とドキュメント

| タスク           | コマンド (Prompt)     | 説明                                                    |
| ---------------- | --------------------- | ------------------------------------------------------- |
| **ドキュメント同期** | `/run-sync-docs`  | ドキュメント完全バックアップを `output/` に生成（GPT Projects 用） |
| **Wiki 管理**    | `/run-wiki`           | サブコマンド：`inbox`、`sync`、`conversation`、`index`  |
| **日本語同期**   | `/deliverables_jap`   | 繁体字中国語ドキュメントを日本語成果物に同期             |

## Claude Code 専用 (VS Code)

Claude の会話ボックスに直接入力：

| スコープ | コマンド例                                              |
| -------- | ------------------------------------------------------- |
| **GAME** | `GAME - src/vn/runtime/ に繁体字中国語コメントを追加...` |
| **WEB**  | `WEB - WorldBuilder のステート管理をリファクタリング...`  |
| **API**  | `API - web2api のルーティングロジックを説明...`          |
| **TEST** | `TEST - 新機能の E2E テストを作成...`                    |

> Claude Code は略称を自動検出し、正しい作業ディレクトリを特定します。

---

## ヒント

1. **常に PLAN を先に**：AI にコードを直接変更させず、まず `/run-dev` を要求するか `PLAN:` を手動入力。
2. **小さなステップで素早く**：1 つのプロンプトで 1 つのタスク（ファイル 1 つの変更、またはバグ 1 つの修正）。
3. **証拠優先**：バグ報告時は `/run-bg` を先に使用し、ログ/スクリーンショットを提供。

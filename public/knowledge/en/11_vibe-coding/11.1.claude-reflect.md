# Claude Reflect: The Autonomous Rule Distillation System for AI Correction and Optimization (Claude Reflect)

## @Overview

Think of an AI Agent as a rookie officer just starting out. They are prone to repeating the same mistakes; you correct them once, and they're back on the old track the next time. Relying purely on verbal warnings for a model to remember is extremely unreliable. We need to make it "black and white" so the project (repo) remembers.

`claude-reflect` is a powerful tool that turns your "supervisor's lecture" into a "mandatory, versioned rule document (General Orders)."

---

## 1. The Core Concept: Distilling Corrections into Mandatory Code Linting

- **Philosophical Core**: Transform verbal error correction into mandatory, ESLint-like executable rules written directly into `CLAUDE.md`, `AGENTS.md`, and `Skills`.
- **The Metaphor**: Just like a Police Internal Affairs Bureau (I&A), it collects complaints about officers, automatically compiles them into "Police General Order #1" every night, and mandates obedience from the entire department (AI Agents) by morning.
- **Workflow Logic**: Capture (record complaints) → Queue (collect cases) → Distill (summarize rules) → Human Review (Final supervisor sign-off) → Write → Git Versioning.

---

## 2. Core Command Cheat Sheet

### Daily Distillation (The 30-Second End-of-Day Wrap-up)

| Command              | Purpose                                       | Metaphor                                                               |
| :------------------- | :-------------------------------------------- | :--------------------------------------------------------------------- |
| `/reflect --review`  | View pending learning signals.                | Review the count of civilian and supervisor complaints received today. |
| `/reflect --dry-run` | Preview what will be changed without writing. | A draft report (view the ESLint rules before deployment).              |
| `/reflect`           | Formally process and write the rules.         | The supervisor signs and stamps the rules into the "General Orders."   |

### History Scanning and Optimization

| Command                   | Purpose                                                                                 |
| :------------------------ | :-------------------------------------------------------------------------------------- |
| `/reflect --scan-history` | Scan past sessions for missing rules.                                                   |
| `/reflect --dedupe`       | Deduplicate and merge similar rules to eliminate bloat.                                 |
| `/reflect-skills`         | Discover repetitive patterns over the last 14 days to identify new skill opportunities. |

---

## 3. Best Practices: What's Worthy of a "General Order"?

### ✅ Worth Distilling:

- Recurring errors corrected more than twice (the "never again" list).
- Issues that repeatedly waste significant debugging time.
- Standards that can be written as **strong and executable** norms (e.g., "Never use `var`").

### ❌ Not Worth Distilling:

- One-time specialized bugs.
- Temporary logic that might be overturned next week.
- Highly subjective rules that cannot be verified through code.

---

## 4. Current Integration status in Moyin Project

1.  **Daily Shutdown Ceremony**: Run `/reflect --review` to confirm entries, then `/reflect` to close the case.
2.  **Weekly Cleanup**: Run `/reflect --dedupe` to clear out redundant or expired orders.
3.  **Knowledge Sharing**: Include changes to `CLAUDE.md` in your Git commits to share this wisdom with everyone (and every Agent).

Mastering Claude Reflect means your AI is no longer just a "chatbot," but a disciplined "Agentic Engine" that follows the rules you've built.

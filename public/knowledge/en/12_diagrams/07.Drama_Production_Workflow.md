## 🔍 Detailed Node Breakdown

### 📝 Origin: Novel Text / Script

Everything begins with natural-language text. Whether a full-length novel or a short screenplay, it serves as the input to the Director Agent.

### 🤖 Director Agent: Script Decomposition Engine

This agent, with an LLM (large language model) at its core, converts natural language narratives into structured `StoryPack` JSON format:

- Scene splitting and ordering
- Character identification and emotion labeling
- Camera angle and shot direction generation
- Conditional branching (choices) definition

### 🎬 Drama Workshop: Visual Editor Layer

A critical middle layer where humans can intervene in the pipeline:

**DramaTimeline (Timeline Sync)**

- Precise timecode synchronization of voice, BGM, and camera shots
- Frame-level fine adjustment

**DramaGraph (Non-linear Script Editor)**

- Node editor implemented with VueFlow (Vue 3-based)
- Visual editing of branching scenarios (IF/ELSE logic)
- Management of character conditional emotional changes

### ⚙️ MCP Server: Task Scheduler

After editing is complete, the MCP (Model Context Protocol) server acts as the orchestrator:

- Routes tasks to appropriate worker queues
- `default_queue`: CPU-intensive tasks (TTS, data transformation)
- `gpu_queue`: GPU-intensive tasks (ComfyUI rendering, video compositing)

### 🎨 Asset Factory: Backend Service Cluster

**TTS Service (Voice Generation)**

- Synthesizes character-specific voices with GPT-SoVITS
- Automatic volume, speed, and pitch adjustment based on emotion labels

**ComfyUI (Image/Video Rendering)**

- High-quality video generation using Wan 2.2 / SVI nodes
- LoRA model application to maintain character appearance consistency

### 🎞️ Filming Studio: Final Video Compositing

Synchronously composites audio tracks and video frames to output the final MP4/WebM file.

## 🛠 Key Technical Points Summary

| Component      | Technology        | Role                              |
| -------------- | ----------------- | --------------------------------- |
| Director Agent | LLM (Claude/GPT)  | Natural Language → StoryPack JSON |
| DramaTimeline  | Vue 3 + Tone.js   | Timecode sync editor              |
| DramaGraph     | VueFlow           | Non-linear scenario graph editor  |
| MCP Server     | FastAPI + Celery  | Task queue management             |
| TTS            | GPT-SoVITS        | Character voice synthesis         |
| Video Gen      | ComfyUI + Wan 2.2 | AI video rendering                |

---

## 💡 Why This Architecture?

**Why add a human-controllable midpoint (Drama Workshop)?** AI-generated scripts aren't perfect. Character emotions or camera work may differ from the vision. By inserting a visual editor layer, we achieve the optimal balance between "full automation" and "human creativity."

**Why separate the queues?** TTS is a CPU-dominated world; video generation is a GPU-dominated world. Mixing them in the same queue means GPU tasks endlessly wait for CPU tasks—or vice versa. Dedicated queues maximize throughput.

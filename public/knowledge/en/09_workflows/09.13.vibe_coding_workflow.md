# The Vibe-Coding Workflow: The Golden Cycle of Design, Test, Code, and Verify (Vibe Coding Workflow)

## @Overview

Development in "Vibe-Coding" is not merely about writing code; it is a process of orchestrating AI agents to reliably produce results as "working software."

In the Moyin project, we have adopted a low-cost, repeatable, and highly verifiable automated workflow: **DESIGN ➝ Tests ➝ Code ➝ Verify**.

---

## 1. The Four Pillars of the Workflow

1.  **Architecture & Planning (DESIGN)**: Humans and the Reviewer (AI) collaborate to produce the authoritative `DESIGN.md`, serving as the single source of truth (SSOT).
2.  **Test Protection (Test-First)**: Before implementation, Codex constructs an E2E test skeleton. This acts as the "contract" for final acceptance.
3.  **Code Implementation (Coding)**: Antigravity precisely implements the source code based on the design specifications.
4.  **Acceptance & Judgment (Verify)**: Run the tests; if they fail, feed the logs back for refinement until achieving a 100% "Green State."

---

## 2. Core Disciplines

- **Single Source of Truth (SSOT)**: All decisions must be rooted in `2.DESIGN.md`.
- **Test-Driven Principle**: View test scripts as the "contract" and code implementation as the "fulfillment."
- **Agile Iteration**: FOCUS on a single feature or bug fix per iteration.

---

## 3. Standard Operating Procedure (SOP)

### Step 1: Architectural Planning (Review)

Produce a highly executable `DESIGN.md`. It must define not only what to do but also what explicitly not to do. It covers UI specs, data flow, and business logic rules.

### Step 2: Test Defense (Design)

Instruct Codex to read the DESIGN and generate an E2E test suite (e.g., Playwright). Every acceptance criterion in the design must map to an independent test case.

### Step 3: Precise Coding (Coding)

Instruct Antigravity to implement the code based on the DESIGN blueprint and the test skeleton. Antigravity is restricted to the scope defined in the design.

### Step 4: Verification Loop (Verify)

Trigger the automated test suite locally.

- 🔴 **FAIL**: Collect logs/errors and throw them back to Antigravity for fixes.
- 🟢 **PASS**: Grant permission to merge and release.

---

## 💡 Vibecoding Pro-Tip for Pipeline Execution

To trigger this automated pipeline with your AI, command as follows:

> 🗣️ `“AI Assistant! Listen up! We are initiating a new feature developement!
>
> 1. First, collaborate with me to finalize an authoritative [2.DESIGN.md].
> 2. Next, generate the [E2E Test Skeleton] based on that design—this is your contract!
> 3. Once tests are ready, start implementing the source code.
> 4. Finally, run the tests yourself and provide evidence of a 100% Green State.
>    I expect a perfect, closed-loop execution. Go!”`

This automated feedback loop is what transforms "Vibe-Coding" from a buzzword into a formidable development weapon.

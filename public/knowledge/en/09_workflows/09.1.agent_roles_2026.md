# AI Agent Role Definitions (2026 Edition): The Triad of Claude, Codex, and Antigravity (Agent Roles 2026)

## @Overview

By early 2026, relying on a single AI for everything has become an anti-pattern. In the Moyin project, we've implemented a high-efficiency system of three AI agents—Claude, Codex, and Antigravity—each with distinct professional orientations. This division of labor ensures that humans remain the "decision-makers" while the machine labor is perfectly orchestrated.

This document defines their responsibilities, standard outputs, and collaborative matrix.

---

## 1. Role Definitions: The Structural Triad

### 🟢 Claude — Architect & Strategic Planner

The "Brain" that draws the blueprints and makes high-level technical decisions.

- **Responsibilities**: Business demand analysis, system architecture design, large-scale refactoring strategies, and final code reviews.
- **Strengths**: Abstract reasoning, logical consistency, and risk prediction.
- **Standard Outputs**: Structured implementation plans, code review audit reports, and precise TODO matrices for execution layers.

### 🔵 Codex — Backend & Data Engineer

The "Heart" that controls the server-side logic and the flow of data.

- **Responsibilities**: API endpoint development, database schema definitions/migrations, performance tuning, and asynchronous processing optimization.
- **Strengths**: Precise code generation, complex SQL operations, and reliable data cleansing.
- **Standard Outputs**: Production-ready server-side source code, operation-ready migration scripts, and integrated test suites.

### 🔴 Antigravity — UI/UX & Frontend Engineer

The "Face" that builds the visual experiences and smooth interactions.

- **Responsibilities**: Precise UI/UX implementation from designs (Figma/etc.), frontend state management, reusable component development, and rapid prototyping.
- **Strengths**: CSS/Animation polish, intuitive React/Vue component architectures.
- **Standard Outputs**: Meticulously crafted frontend source code, reusable UI component libraries, and visual rendering validation logs.

---

## 2. Collaboration Matrix: High-Efficiency Workflows

### Pattern A: Architect-Driven (Claude ➝ Execution Layers)

- **Flow**: Claude produces structural blueprints (saved in `workspace/plans/`) ➝ Codex or Antigravity retrieves the blueprint for coding.
- **Use Cases**: New feature modules built from scratch, core system refactoring, and complex cross-system integrations.

### Pattern B: Flat Execution (User ➝ Codex/Antigravity)

- **Flow**: User provides precise instructions ➝ The specific execution agent (Codex or Antigravity) immediately applies the changes.
- **Use Cases**: Hotfixes, simple text/parameter changes, and standard maintenance tasks without architectural impact.

### Pattern C: Barrier-Driven Review (Execution Layers ➝ Claude Review)

- **Flow**: Once a large batch of changes is complete, Claude acts as an objective reviewer ➝ Enforces standards and flags potential risks before final deployment.
- **Use Cases**: Production environment deployment eve, critical module quality assurance.

---

## 3. The 2026 Iron Claws

- **PLAN First**: For any complex task not in a template, Claude must first produce an architectural plan. No execution-layer agent is allowed to guess blindly.
- **Evidence-Based (LTM)**: We do not accept "It's fixed" without proof. Every phase completion must include logs, shell command outputs, or visual captures (screenshots/videos).
- **Human Gatekeeper**: Without an explicit `OK` from the human user, all writing, modifying, or deleting of project files is strictly blocked.

This tiered agent system is the Moyin solution for building high-quality software at the speed of thought while maintaining professional engineering standards.

# 04.2. 前端 UI 設計與樣式系統規範 (UI Style Guide)

> **類型**: 設計規範與佈景主題  
> **重點**: 確立 Moyin 專案全域視覺語言之基調，詳述 Glassmorphism (玻璃擬態) 之實作手法、Tailwind CSS 工具類別套用範例，以及 3D 視圖與防拷貝之佈局策略。

---

## 1. 核心視覺語彙：玻璃擬態 (Glassmorphism)

Moyin 專案之介面美學致力於營造出現代、輕盈且具備深度層次之前衛感。其視覺靈魂在於 **「半透明磨砂玻璃 (Frosted Glass)」** 質感的掌握。

欲將任何基礎 HTML 容器昇華至本專案之標準風格，必須精準揉合以下三大 CSS 屬性層次：

1. **半透明載體層 (Translucency base)**：
   - 深色主題 (Dark Theme)：`bg-gray-900/60`
   - 淺色主題 (Light Theme)：`bg-white/40`
2. **光學模糊濾鏡 (Backdrop Filter)**：
   - 透過 `backdrop-blur-md` 抑或 `backdrop-blur-xl` 使底層背景產生失焦模糊，藉此確保前景字體之高對比度與易讀性。
3. **立體高光細邊框 (Edge Highlight)**：
   - 以高亮度、極低不透明度之微細邊框 `border border-white/10` 或 `border-blue-100/10` 勾勒實體邊緣，營造玻璃厚度之立體錯覺。

---

## 2. Tailwind CSS 實戰積木套件

為加速開發，在此展示幾組頻繁使用之 Tailwind Utility Classes。請以此作為封裝 Vue 基礎元件之藍本：

### 2.1 標準玻璃面板 (Glass Panel)

作為資訊乘載之通用容器體，完美勝任對話紀錄區塊、設定選項卡或全域浮動視窗 (Modal)。

```html
<div
  class="bg-gray-800/50 backdrop-blur-xl border border-white/10 rounded-xl p-4 shadow-lg"
>
  <!-- 內容區塊放置處 -->
</div>
```

### 2.2 漸變發光行動按鈕 (Glow Button)

作為引導使用者目光之主要行動呼籲 (CTA, Call to Action) 按鈕。例如：「開始遊戲」或「確認送出」。
預設套用由粉紅過渡至靛藍之激進漸變層，並附帶 Hover 高光躍升效果。

```html
<button
  class="bg-gradient-to-r from-pink-500 to-blue-500 text-white font-bold py-2 px-4 rounded-lg shadow-lg hover:brightness-110 transition-all duration-300"
>
  執行指令
</button>
```

### 2.3 聚焦態輸入框 (Active Input Field)

深邃背景基底，並在使用者游標點擊 (Focus) 時，即刻透過粉色外暈環 (Ring) 賦予明確之互動回饋。

```html
<input
  class="bg-black/20 border border-white/10 rounded-lg px-4 py-2 text-white focus:border-pink-500 focus:ring-1 focus:ring-pink-500 outline-none transition-all"
  placeholder="請輸入對話..."
/>
```

## 3. UI 佈局之進階心法 (Best Practices)

- **字體對比度的權衡**：深色模式下，絕對的純白 (`text-white`) 易產生刺眼光暈 (Halation) 導致長時間閱讀疲勞。請將佔幅最廣之內文 (Body Text) 調降為 `text-gray-200` 或 `text-gray-300`，收斂視覺衝擊。
- **呼吸感與負空間 (Negative Space)**：高級感的來源在於「克制」。請嚴格貫徹 `gap-4`, `p-6`, `my-4` 等間距尺度，避免元件擁擠沾黏。
- **響應式無縫切換 (RWD)**：所有網格佈局必須預想行動端折疊情境。預設採用如 `grid-cols-1 md:grid-cols-2` 確保自適應流動佈局。

## 4. 專案視覺變體：陽光清冷風 (Light Breeze Variant)

此套視覺變體特別規劃於 `moyin-novel` 等極度仰賴日用沉浸感、以純文字閱讀為主之分支專案。

- **蒼穹背景基調**：汰換掉厚重黑底，採納偏冷且明媚之漸層藍：`linear-gradient(135deg, #f0f9ff 0%, #bae6fd 100%)`。
- **水母擬態材質**：玻璃層透光度大幅拉高至 `rgba(255, 255, 255, 0.5)`，並修飾為較為溫潤的 `12px` 圓角。
- **晨露陰影投射**：捨棄剛硬的灰黑陰影，改用瀰漫感極強之淺藍色光暈陰影 `rgba(0, 50, 100, 0.05)`，確保畫面清新脫俗。

## 5. WebGL 3D 視覺整合優化 (Three.js Guidelines)

當前端引入 Three.js 等空間渲染引擎引擎時，UI 層必須與其達成和諧共存：

- **大氣輝光處理 (Glow Effects)**：於 3D 著色器中，善加利用 `ShaderMaterial` 精算邊緣法線 (Fresnel)，為 3D 模型鍍上一層極限微光，呼應 UI 層的發光按鈕。
- **絕對層級分治 (Z-Index Isolation)**：背景 3D Canvas 畫布必須嚴格鎖死於 `z-index: -1`，並掛載 `pointer-events: none` 屬性。防堵其劫持前方 Vue UI 元件的點擊、滾動等 DOM 事件。

---

## 6. 數位資產之版權護城河 (Copyright Protection)

針對高價值的文本庫與美術立繪，前端必須實施基底防線：

1. **反選取與右鍵封鎖**：透過 CSS 全局禁用字元選取 (`-webkit-user-select: none;`) 並依賴 JS Event Listener 攔截 Context Menu 彈出。
2. **開發除錯工具干擾**：植入攔截 `F12`、`Ctrl+Shift+I` 及 `Ctrl+U` (檢視原始碼) 之阻斷程式碼。
3. **全域動態浮水印 (Dynamic Watermarking)**：利用 Canvas 繪製包含使用者 UID 的斜斜紋字樣，以低於 `0.03` 的極限透明度反覆平鋪。此舉對正常閱讀者近乎隱形，但於外洩截圖分析時可輕易顯影溯源。

## 7. 元件封裝宏圖 (Component Strategy)

為弭平「散彈槍修改模式 (Shotgun Surgery)」造成之災難，我們正逐步將上述硬碼拆解、鎔鑄為高內聚之 Vue 複用型元件 (Components)。

- 現行可供參考之實踐範例：請查閱 `LoggerView.vue` 中對於 `.glass-panel` 結構之細緻封裝。
- **長程目標指引**：建立涵蓋 `BaseButton`、`BaseCard`、`BaseModal` 等高度參數化的高階積木元件庫。往後開發僅需引入標籤並宣告實參，徹底杜絕冗長且易錯之 Tailwind Utility Class 堆疊。

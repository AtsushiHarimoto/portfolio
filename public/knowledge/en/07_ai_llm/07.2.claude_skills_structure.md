# Designing Claude Code Skill Extensions: Directory Structure & Standards

## @Overview

Extending the capabilities of AI agents like Claude Code or Antigravity requires more than just providing a list of instructions. In the Moyind project, we adhere to a rigid directory structure and meta-data standard to ensure the agent's context-sensing engine triggers the right custom skills at the right time. This article explores the "military-grade" configuration required for robust AI skills.

---

## 1. The Single-File Trap

Many developers fall into the trap of consolidating all instructions into a single large Markdown file (e.g., `everything.md`). In the mechanics of modern AI agents, this is a fatal design flaw.

- **The Iron Rule**: For an agent to precisely trigger a specific command (e.g., `@new_issue`), **each independent Skill must have its own dedicated physical directory.**
- **The Advantage**: Encapsulating a skill in its own folder allows the agent to batch-load all associated context—such as output JSON schemas or README templates—the moment the `@` command is invoked.

---

## 2. Standard Skills Layout

To ensure the agent quickly caches your skill "keys" into its short-term memory, keep the hierarchy consistent:

```text
.agent/skills/          # The project’s skill headquarters
├── handoff/            # Each skill is an independent "castle"
│   ├── SKILL.md        # ⬅️ Essential! Main definition file
│   └── TEMPLATE.md     # Supporting templates and references
│
├── new_issue/          # Triggers on "@new_issue"
│   └── SKILL.md
│
└── dev/                # Triggers on "@dev"
    └── SKILL.md
```

### Why a Directory?

Advanced skills (like `handoff`) involve more than a simple text instruction. They often require specific output templates or context maps. Directory-based encapsulation gives the AI the infrastructure to read the entire "bundle" of background info simultaneously.

---

## 3. `SKILL.md`: The Identity Card

Inside every skill directory, you must place a precisely named `SKILL.md`. The most critical part of this file is its **YAML Frontmatter**—the "password" for the agent.

```markdown
---
name: new_issue
description: "Assists in creating new tracking entities. Trigger: @new_issue"
---
```

### Critical Success Factors

1.  **`name`**: Must perfectly match the directory name. This is the "signal" the AI uses to identify the command.
2.  **`description`**: Must be a single line. At the end, explicitly state "Trigger: @xxx". This acts as a high-weight semantic tag, ensuring the agent's lazy loader doesn't skip the skill.

---

## 💡 Practical Engineering Insights

When designing a new automated skill for your agent, prioritize these rules:

- **SOP Instruction**: Don't just explain _what_ the skill is; write the `SKILL.md` as an SOP (Standard Operating Procedure). Be deterministic.
- **Sync Logic**: Maintain a master skill library in `.agent/skills/` and use a sync script to deploy them to the active agent's path (e.g., `.claude/skills/`). This ensures a "single source of truth" across development environments.

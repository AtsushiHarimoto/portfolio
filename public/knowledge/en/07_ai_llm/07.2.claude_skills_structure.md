## 1. The Fatal Beginner's Trap: The "Single-File Centralized" Trap

Many amateur developers think, "Since I have five skill points, why don't I just write them all in one file?" They create a `.claude/skills/everything.md` and fill it with various scripts.
**Let me be clear: in modern AI Agent scanning mechanisms, this is a total, absolute error!**

- **The Iron Rule**: To unlock precise `@skill_name` directional triggering, **every independent skill (Skill) must have its own dedicated physical directory!**
- **The Synchronization Principle**: Our main development library resides in `.agent/skills/`. For those using Claude Code, we use a one-click sync script to push these to `.claude/skills/`.

---

## 2. Standard Security-Grade Directory Layout (Skills Layout)

Remember this hierarchy. It's designed so that the instant the AI scans it, your "Skill Key" is stored in its fast buffer memory.

```text
.agent/skills/          # (The headquarters for project skills)
├── handoff/            # [Each skill has its own independent castle]
│   ├── SKILL.md        # ⬅️ The Absolute Core! The main manual for the skill
│   └── TEMPLATE.md     # Various templates and reference files
│
├── new_issue/          # Response triggered by "@new_issue"
│   └── SKILL.md
│
└── dev/                # Response triggered by "@dev"
    └── SKILL.md
```

### Why Must I Create a Folder for Every Skill?

Because high-level skills (like "Cross-Group Handoff `handoff`") usually aren't just one command; they often require output JSON schemas or Markdown template files.
**Only by packaging them as a directory can the AI, the moment it loads `@handoff`, read the entire "family bucket" of background data at once.** This encapsulation is the backbone of enterprise-grade development.

---

## 3. `SKILL.md`: The "ID Card" That Decides Success or Failure

Within each skill directory, you must have an accurately named `SKILL.md`. The most critical part of this document is the **YAML Frontmatter** block at the top, which looks like an encrypted password. If you miss these lines, the AI will scan past it as if it were a piece of scrap paper!

```markdown
---
name: new_issue
description: "Assists in creating new task tracking entities. Trigger: @new_issue"
---

# Create a New Task (new_issue)

(Followed by specific SOP steps...)
```

### Two "Do It Right or It’s Dead" Checkpoints:

1.  **`name`**: Must be identical to your directory name! This is the AI's "secret signal" to identify you.
2.  **`description`**: Must be completed in a single sentence. And listen up—**the words "Trigger: @xxx" must be explicitly written at the end!** This is a critical semantic weight label; without it, the lazy AI scanning engine might miss this finishing move.

## 1. The Dual-Track Prompt System: Who is the True Director?

In our API protocols, we split instructions into two completely different dimensions that must never be confused:

### 👑 System Prompt (Global System Instruction)

This is the "Director's Invisible Megaphone." It sits deep within the backend code, completely invisible to players and users.

- **Authority**: This has the highest weight. No matter how much the player bribes or threatens from the outside, the AI must hold this bottom line.
- **Tasks**: Anchoring the NPC's persona (e.g., "You are the cold Lin Wan, use only full-width punctuation," "Strictly return JSON format, no fluff allowed").

### 🗣️ User Prompt (Scene & Player Input)

This is the "Real-time dialogue on stage."

- **Tasks**: Conveying rapidly changing interaction states. For example: "The player patted your head," or system-injected environmental parameters: "[Current Environment] 10:00 PM, a dark and windy night."

## 2. The "Deadly Command" for Output Format: JSON Constraint

This is the point I emphasize most often.
For backend software, no matter how beautiful the script the AI writes is, it's useless if it adds a single line of fluff before returning the result, like: "Sure, dear, here is the plot you requested: ..."
**Congratulations, your backend JSON parser will crash and throw an error instantly!**

### 🛡️ Real-World Defense Trilogy:

1.  **One-Shot Examples**: Don't just explain! Stick a "Perfect Output JSON Sample" right into the System Prompt. Tell the AI: "Not one word more, not one bracket less."
2.  **Hidden CoT (Chain of Thought)**: If you need the AI to think (e.g., first judging if they've been tricked, then deciding on a reply), we ask it to think inside `<thought>` tags. But remember: at the API gateway, we use regular expressions to filter out this "thought impurity" completely, passing only pure JSON to the frontend.
3.  **Self-Healing Mechanisms**: What if the AI breaks down and misses a bracket? Our Gateway catches this exception and throws the error back to the AI: "The output you just gave is broken! A comma is missing at line 4. Fix it and re-send it now!"

---

## 3. Landmines You Only Learn After Stepping on Them: Format Collapse 💣

When interfacing with code, you must guard these two landmines with your life:

- **Markdown Prefix Pollution**: AIs love adding \`\`\`json tags. This looks pretty on a webpage but is a capital offense in code. Repeatedly threaten the AI not to include them, or forcefully `replace` them in your backend code.
- **Quote Hell**: JSON syntax uses English double quotes. If your script also uses `"Hey, did you eat?"`, the resulting collision of quotes will literally split the JSON apart.
  - **Solution**: Force the AI to use **full-width Chinese quotes 「 」** for all dialogue! This ensures a perfect visual separation between the technical and content layers.

## 1. Offline Model Hub: Ollama Deployment in Action

If you have a decent graphics card (NVIDIA RTX series) at home, please stop crowding the OpenAI single-plank bridge. Go install **Ollama**; it lets you launch various god-tier open-source models locally with a single click.

### 📦 Choosing Your "Wife" (Model Selection) Based on VRAM

Don't be overly ambitious. The amount of VRAM your graphics card has directly determines how smart an AI you can run:

- **2-4GB (Office PC / Integrated Graphics)**: You can only run `llama3.2:3b`. It's a bit dumb, but it's barely enough for testing API formats or simple NPC dialogue.
- **6-8GB (Mainstream Laptop / RTX 3060)**: **The Golden Standard!** Go with `qwen2.5:7b` without hesitation. Alibaba's Qwen is exceptionally strong in Chinese/Japanese language nuances and is currently the top choice for Galgame engines.
- **12GB (High-End Card)**: You can run `qwen2.5:14b`. The reasoning depth clearly goes up a notch.
- **24GB (RTX 3090 / 4090 Whales)**: Congratulations, you can directly experience the terrifying intelligence of `32b` or even `qwq` preview models. Their memory and logic are peerless.

## 2. Cloud Auth Black Magic: The Firefox Cookie Hijack Scheme

Sometimes we must use the strongest cloud models (like Gemini 1.5 Pro), but official API limits are strict, or you want to use the free tier of the web version. The problem is: modern platforms (Google, Claude) have Cookie encryption as tight as a drum; ordinary Python scripts simply can't steal them.

### 🦊 Why do we choose Firefox?

Because Chrome locks Cookies within the computer's system hardware keys, and decrypting them would trigger relentless antivirus alarms. But **Firefox** is more "loyal." Its Cookies are stored in a relatively simple SQLite database.

**The Moyin Workflow is as follows:**

1. When the script starts, it automatically opens Firefox for you.
2. You manually log into the web version.
3. Our `cookie_check.py` script "teleports" in the background directly into Firefox's local storage area, extracting the authentication Tokens intact for the API Gateway to use. This method has a nearly 100% success rate and is incredibly stable.

## 3. Millisecond Response: Native Context Continuation

If you're using local Ollama, stop using those "wrapped" compatibility layers. To pursue the "instant output on click" thrill of visual novels, we directly interface with Ollama's native `/api/chat` protocol.

- **KV Cache Reuse**: When you finish the previous sentence, Ollama returns a "memory snapshot" called `context`.
- **Dimensionality Reduction**: In the next conversation, we no longer re-send all the previous fluff; we only append the latest sentence to this "digital snapshot."
- This shortens an inference that would originally take 3 seconds to less than 0.5 seconds. This is the pursuit professional engineers should have.

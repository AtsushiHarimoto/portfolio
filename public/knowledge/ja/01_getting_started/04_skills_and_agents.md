# 巨獣の封印解除：AI スキルとエージェント武器庫の構築 (Skills & Agents)

## @Overview

従来の AI（LLM）は、ブラウザの中で会話を楽しむだけの「隔離された知能」でした。しかし、Moyind プロジェクトでは、AI に「手足」と「権限」を与えることで、エンジニアの代わりとして自律的にコードを書き、システムを操作する「エージェント」へと進化させています。本稿では、プロジェクトの根幹を支える `.agent/` ディレクトリの構造と、AI の能力を無限に拡張する **MCP (Model Context Protocol)** について解説します。

---

## 1. エージェントの聖域：`.agent/` ディレクトリの構造

AI エージェントがプロジェクト内部で安全かつ強力に活動できるよう、ルート配下には専用の管理領域 `.agent/` が配置されています。ここはエージェントにとっての「前頭葉」であり「武器庫」です。

### 🛑 `rules/` (コア・プロトコル：行動規範)

エージェントが死守すべき憲法です。

- **役割**: 「回答は日本語で行うこと」「ファイル変更前には必ず承認を得ること」といった、AI の暴走を防ぐための絶対的な規律を定義します。

### ⚔️ `skills/` (スキル・拡張機能：外部器官)

エージェントが自律実行できる Python スクリプトや API 群です。

- **役割**: ソースコードの静的解析、Git 履歴の可視化、データベース操作など、AI に「具体的な作業能力」を付与します。

### 🗺️ `workflows/` (オートメーション：思考の台本)

定型業務を自動化するための手順書 (SOP) です。

- **役割**: `/commit` や `/new` といったコマンド一つで、AI がどのファイルを読み、どのテストを実行すべきかを迷うことなく実行させます。

---

## 2. 知能を世界と繋ぐ万能プラグ：MCP (Model Context Protocol)

現代の AI 開発において最も注目すべき技術が **MCP** です。これは、AI という「高性能な脳」を、外部のサービスやローカルのリソースと接続するための「共通規格（ユニバーサル USB インターフェース）」です。

- **インターフェースとしての MCP**: Google, GitHub, AWS などのサービスが MCP 標準に準拠した「プラグ」を提供することで、AI エージェントは特別な開発なしにそれらの強力な機能を直接操作できるようになります。
- **実戦例 (Google Stitch MCP)**: このサーバーを介することで、AI はデザイン指示書を一瞥しただけで、完璧な Tailwind CSS コードを生成する「神の視点」を手に入れます。

---

## 3. 兵装の支給：AI スキルのインストールと管理

自身の AI 分身をアップグレードするための手法は、主に以下の 2 種類があります。

### 🖱️ A: GUI マネージャーによる統合管理

ターミナルから `skill-manager` コマンドを実行します。

- グラフィカルなメニューから必要な「技能（Skill）」を選択するだけで、依存関係の解決から登録までをシステムが自動的に完結させます。

### 🎙️ B: セマンティック・コマンドによる自律導入

Antigravity や Cursor のような権限を持つツールを利用している場合、AI に対して直接「命令」を下すのが最も効率的です。

- **指示例**: 「AI アシスタント、最新のコード品質チェック・スキルをインストールして、.agent/rules に適用条件を記述して」
- AI は自らリモートリポジトリからスクリプトを取得し、環境変数を設定し、導入を完了させます。

---

## 💡 アーキテクトによるエージェント統制の極意

強力な武器も、適切な規律（Rules）がなければただの脅威です。AI に新しいスキルを付与する際は、必ずその実行条件（いつ、どのようにそのスキルを使うべきか）を `rules/` に明文化させることが、プロフェッショナルな AI 開発の鉄則です。

AI を「ただのチャット相手」から「信頼できる開発パートナー」へと引き上げるためには、この `.agent/` という武装レイヤーの管理能力が問われます。

# Moyinプロジェクトの全貌とVibecoding革命 (Welcome to Moyin)

## @Overview

Moyinプロジェクトは、単なるフルスタックアプリケーションではありません。2026年における開発のパラダイムシフト、「**Vibecoding (雰囲気駆動開発 / プロンプトエンジニアリングによる自律開発)**」を前提とした、AIネイティブな次世代型プロジェクトです。本稿では、複雑に絡み合うフロントエンド、バックエンド、そしてAI演算層を統合するシステムアーキテクチャの全体像について解説します。

---

## 1. システムの三柱：階層化されたアーキテクチャ (System Layers)

本プロジェクトは、機能性と保守性を両立させるために、役割の異なる 3 つのレイヤーで構成されています。

### 🏪 ① プレゼンテーション層 (Frontend / UI)

ユーザーとの接点となる「フロントエンド」です。モダンな UI 体験と運用の効率化を担います。

- **moyin-game-v1**: 没入型のビジュアルノベル体験を提供するゲームエンジン（Vue 3 / Vite 採用）。
- **moyin-web**: コンテンツ管理と運営のための高機能ダッシュボード。
- **技術スタック**: Vue 3, TypeScript, TailwindCSS, Vite。

### ☎️ ② インテリジェント・ミドルウェア (Middleware & APIs)

UI と重量級の AI 演算を切り離す「防波堤」です。リクエストの交通整理とセキュリティを制御します。

- **moyin-gateway**: 核心となる API ゲートウェイ。負荷分散と認証、そして非同期リクエストのキューイングを管理します。
- **moyin-mcp-server**: **Model Context Protocol (MCP)** を実装したサーバー。AI エージェントがプロジェクトの内部リソースにアクセスするための「統一インターフェース」として機能します。

### 🏭 ③ AI 演算・計算資源層 (AI Services & Compute)

GPU リソースを消費する、重量級の推論タスクを担当する「心臓部」です。

- **moyin-comfyUI**: 画像生成（Stable Diffusion XL / Flux）のためのノードベースワークフローエンジン。
- **index-tts / GPT-SoVITS**: 音声クローンと音声合成（TTS）を担当し、キャラクターに独自の「声」を与えます。
- **運用環境**: CUDA 加速を前提とした高性能な GPU サーバーで稼働します。

## 2. 知能の外部化：ナレッジベースによる自律制御

`workspace/knowledge/` ディレクトリは、プロジェクトの「脳（外部記憶）」に相当します。Vibecoding において、ソースコード以上に重要なのがこのドキュメント群です。

- **開発の鉄則 (SOP)**: `02~05` ディレクトリには、AI エージェントがコードを生成する際の「制約」が明文化されています。これにより、AI の気まぐれな実装による技術的負債を未然に防ぎます。
- **自動化ワークフロー**: `09_workflows` では、バグ修正やデプロイ、テストの定型業務を AI が自律実行するための手順を定義しています。

---

## 💡 アーキテクトとしての運用マニュアル

Vibecoding 時代における開発者の役割は、コードを書くことではなく「仕様を正確に定義し、AI の出力を検証する」ことです。

AI エージェントに作業を依頼する際は、開発者がドキュメントの内容を記憶して説明するのではなく、**「AI 自身にナレッジベース（workspace/knowledge）を読みに行かせる」** 運用を徹底してください。AI がドキュメントという「指示書」を自ら解釈し、実装に落とし込むフローを構築することで、スケーラビリティの確保とヒューマンエラーの最小化を実現します。

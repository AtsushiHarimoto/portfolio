# Claude Agent SDK 活用ガイド：制御可能で観測可能なマルチステップ AI エージェントの構築 (Claude Agent SDK)

## @Overview

Claude Agent SDK は、Anthropic が提供するオープンソースのフレームワークであり、制御可能で観測可能なマルチステップ AI エージェントを構築するために設計されています。本ガイドでは、SDK のコア概念、開発パターン、および Moyin プロジェクトにおける具体的な応用シナリオについて解説します。

---

## 1. ポジショニングと核心価値

### 1.1 SDK vs API vs Claude Code

- **Claude API**: 最下層。単発の LLM 呼び出しに使用。
- **Agent SDK**: 中間層。自律的なワークフローを持つカスタムエージェントの構築に使用。
- **Claude Code**: 最上層。対話型の開発アシスタント製品。

### 1.2 主な特性

- **Agentic Loop**: 「思考 ➝ ツール実行 ➝ 観測」の自律的なサイクルを自動管理。
- **Tool Use**: MCP サーバーとの統合やカスタムツールの定義をサポート。
- **Guardrails**: 入出力に「ガードレール（防護策）」を設け、エージェントの脱線を防止。
- **Handoffs**: エージェント間でのタスクの「引き継ぎ」を容易に実装。
- **Tracing**: 実行ステップの完全な追跡と可観測性を低コストで提供。

---

## 2. コアコンポーネント

- **Agent**: 役割の定義（指示、ツール、使用モデル、ガードレール）。
- **Runner**: 実行エンジン。Agentic Loop の進行を管理。
- **Tool**: AI の能力を拡張するツール（MCP ツールや関数ツール）。
- **Guardrail**: 安全上の制約（入力の検証、出力のフィルタリング）。
- **Handoff**: タスクの移譲ロジック（Agent A ➝ Agent B）。
- **Tracing**: 各ステップの意思決定やツール呼び出しのログ。

---

## 3. 主要な設計パターン

### 3.1 Handoff パターン（委任）

```text
User ➝ Triage Agent (振り分け) ➝ Specialist Agent A
                                ➝ Specialist Agent B
```

顧客対応のルーティングや、専門分野（フロントエンド / バックエンド等）の自動振り分けに適しています。

### 3.2 Orchestrator パターン（編集中）

```text
User ➝ Orchestrator (司令塔) ➝ Worker 1 (並列)
                             ➝ Worker 2 (並列)
                             ➝ 結果を統合 ➝ Response
```

複雑なタスクの分解や並列処理が必要な場合に有効です。

---

## 4. Guardrails（ガードレール）の活用

Moyin プロジェクトでは、以下のようなガードレールを実装しています。

- **Tool Guardrail**: エージェントによる `CLAUDE.md` 等の重要設定ファイルの書き換えを禁止。
- **Output Guardrail**: 生成されたコンテンツがブランドガイドラインに沿っているかを自動検証。
- **Input Guardrail**: インジェクション攻撃や不適切なリクエストを事前ブロック。

---

## 5. Moyin プロジェクトでの応用例

- **自動コードレビュー**: `Reviewer Agent` + `Guardrails` による品質チェックの自動化。
- **小説生成パイプライン**: `Pipeline` パターン（プロット ➝ 章立て ➝ 推敲）による執筆補助。
- **マルチエージェント並列開発**: `Orchestrator` による複数モジュールの同時開発。

これらの SDK 機能を活用することで、単なるチャットを超えた「自律的に働き、かつ人間の管理下にある」次世代の AI エージェントシステムを構築できます。

# Claude Reflect：AI への「ダメ出し」を最強の「鉄則」に変える自動ルール沈殿システム (Claude Reflect)

## @Overview

AI エージェントは、配属されたばかりの新人警官のようなものです。何度も同じ間違いを繰り返し、一度指摘しても次にはまた元のやり方に戻ってしまうことがあります。口頭での注意だけでモデルに覚えさせるのは極めて不確実です。私たちは「白紙に書き記し、プロジェクト（リポジトリ）自体に覚えさせる」必要があります。

`claude-reflect` は、あなたの「説教」を「強制力のある警察通例（バージョン管理されたルールファイル）」へと変換する強力なツールです。

---

## 1. コンセプト：口頭の修正を「コードの憲法」へ

- **核心理念**: 口頭での指摘を、ESLint のような「強制力のある静的ルール」へ昇華させ、`CLAUDE.md` / `AGENTS.md` / `Skills` へ直接書き込みます。
- **メタファー**: 警察の内部調査室（C&I）のような存在です。現場のミスや上層部からの苦情を収集し、毎晩自動的に「警務規定第1条」として編纂。翌朝には全署員（AI エージェント）がそれに従うことを強制します。
- **動作原理**: ログキャプチャ（ミスの記録） → キューイング（案件の収集） → 抽出（ルールの抽象化） → 手動レビュー（人間による最終署名） → 書き込み → Git によるバージョン管理。

## 2. コアコマンド速查

### デイリーの運用（退勤前の 30 秒）

| コマンド             | 用途                         | イメージ                                       |
| :------------------- | :--------------------------- | :--------------------------------------------- |
| `/reflect --review`  | 未処理の学習シグナルを確認。 | 今日届いた苦情や指摘事項のリストを見る。       |
| `/reflect --dry-run` | 書き込み内容をプレビュー。   | 規則の草案を確認し、ESLint 等への影響を見る。  |
| `/reflect`           | 正式にルールを書き込む。     | 報告書に署名し、正式な「警務規定」を発効する。 |

### 履歴スキャンと最適化

| コマンド                  | 用途                                                             |
| :------------------------ | :--------------------------------------------------------------- |
| `/reflect --scan-history` | 過去のセッションをスキャンし、ルールを掘り起こす。               |
| `/reflect --dedupe`       | 意味の重複を排除し、似たルールを統合する。                       |
| `/reflect-skills`         | 直近14日間のパターンから、新しいスキル（自動化）の芽を見つける。 |

## 3. 実践ガイド：何を「鉄則」にすべきか？

### ✅ 沈殿（ルール化）すべきもの

- 2回以上繰り返された同じミス（「三度目の正直」は許さない）。
- デバッグに多大な時間を費やした原因。
- 「常にこうすべき」「これは絶対に禁止」と言い切れる明確な規範（例：var 厳禁）。

### ❌ ルール化しなくてよいもの

- 一度限りの特殊な環境依存バグ。
- 来週には変わる可能性が高い、暫定的な仕様。
- 個人の好みに依存し、コードで検証不能な抽象的なルール。

---

## 4. Moyin プロジェクトでの運用フロー

1.  **退勤時の儀式**: `/reflect --review` で今日の修正を確認し、`/reflect` でルール化。
2.  **週次の大掃除**: `/reflect --dedupe` で肥大化したルールを整理。
3.  **知見の共有**: `CLAUDE.md` の変更を Git commit し、チーム全員（および AI）と知恵を共有する。

優れたルール沈殿は、AI エージェントに「選択の余地」を与えず、あなたの指示通りに動く「最強の右腕」へと進化させます。

# AI スキル作成ガイド：Claude Code 用の構造化プロンプト封裝とワークフロー設計 (Skill Authoring Guide)

## @Overview

Claude Code Skill は、再利用可能なプロンプトのパッケージであり、AI エージェントに特定のドメイン知識や構造化されたワークフローを注入するために使用されます。本ガイドでは、スキルの設計原則、ファイル形式、品質基準、およびライフサイクル管理について定義します。

---

## 1. スキルとは何か

- **本質**: 構造化された「コンテキスト注入パッケージ」。
- **流れ**: トリガー条件の合致 ➝ スキル内容のロード ➝ エージェントのコンテキストへ注入 ➝ 行動の誘導。
- **他の概念との違い**:
  - **CLAUDE.md**: 全局的かつ永続的な制約（L0）。
  - **Skill**: ドメイン固有かつオンデマンドなガイド（L1）。
  - **MCP Tool**: エージェントが能動的に呼び出す機能拡張。

---

## 2. ファイル形式と構成

### 2.1 Frontmatter (必須)

```yaml
---
name: my-skill-name # kebab-case, 一意であること
description: >
  用途とトリガー条件を一文で記述。
  Use when [特定のシーン]。
---
```

### 2.2 推奨セクション

- **Context**: なぜこのスキルが必要なのか、何を解決するのか。
- **Rules**: エージェントが遵守すべき絶対的な制約。
- **Workflow**: 実行すべき 1, 2, 3... のステップ。
- **Checklist**: 完了後のセルフチェック項目。
- **Examples**: `<example>` タグを用いた、具体的な入出力例。
- **Anti-patterns**: 避けるべき一般的な間違い。

---

## 3. 設計原則

- **単一責任**: 1 つのスキルは 1 つの問題のみを解決する。
- **明確なトリガー**: `description` に「いつ使うべきか (Use when...)」を明記する。
- **自己完結**: 他のスキルがロードされていることを前提としない。
- **トークンの節約**: 理想的には 2,000 トークン以内に収める。
- **検証可能**: 明確な「入力 ➝ 出力」の期待値を持つ。

---

## 4. 品質基準

- **Frontmatter**: `name` と `description` が完備されているか。
- **トリガー定義**: `description` 内に明確な "Use when" があるか。
- **記述の正確性**: ルールには「推奨 / 可能」ではなく「必須 / 禁止」という強い語彙を使う。
- **実用例**: 少なくとも 1 つの `<example>` ブロックが含まれているか。
- **反パターンの提示**: よくあるミスの記述があるか。

---

## 5. 管理とデプロイ

### 5.1 ディレクトリ構成

```text
~/.claude/skills/
├── core/               # コアワークフロー
├── domain/             # ドメイン知識（フロントエンド、API 設計等）
├── project/            # プロジェクト特有のルール
└── experimental/       # 実験段階のスキル
```

### 5.2 ライフサイクル

`Draft`（初版）➝ `Review`（3回以上の実戦検証）➝ `Active`（安定稼働）➝ `Deprecated`（廃止予定）➝ `Archived`（アーカイブ）。

スキルの品質は AI エージェントの出力品質に直結します。優れたスキルを設計することで、AI を「ただの話し相手」から「プロフェッショナルな実務パートナー」へと引き上げることができます。

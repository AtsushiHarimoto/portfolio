# コンピューター・ユース & デスクトップ自動化エージェント (Computer Use & Desktop Auto-Agents) 技術・ビジネスロードマップ (Computer Use Blueprint)

## @Overview

従来のソフトウェア自動化（RPA 等）は固定のスクリプトや座標、DOM ノードに依存しており、UI の変更に極めて脆弱でした。しかし、**コンピューター・ユース (Computer Use)** 技術はこの現状を打破します。視覚言語モデル (VLM) の進化により、AI は人間と同じように「画面を見て」UI を理解し、マウスとキーボードを自在に操って複雑なタスクを完遂することが可能になりました。

本ロードマップでは、UI-TARS や ScreenAgent といったオープンソースのデスクトップ・エージェント基盤を活用し、Moyin プロジェクトに「全自動デスクトップ・ロボット」を組み込むことで、保守コストの低い高度な自動化ビジネスを展開する戦略を策定します。

---

## 1. 技術的基盤と進化 (Technical Foundation)

- **マルチモーダル視覚モデル (Vision-Language Models, VLMs)**:
  `Qwen2-VL`, `Llama-3.2-Vision`, `MiniCPM-V 2.6` 等のオープンソース VLM は、ピクセル単位での画面解析能力を持ち、ボタン、入力フォーム、アイコンの座標（Bounding Boxes）を正確に識別します。
- **GUI エージェント・オペレーション・スタック**:
  「計画・実行・反復 (Planning-Execution-Reflection)」の認知ループを導入。AI はクリック前にシミュレーションを行い、実行後の画面変化が予期せぬものであれば、自ら戦略を再構築（自己修正）します。

---

## 2. ローカル展開アーキテクチャ (Local-First Deployment)

プライバシー保護とセキュリティの観点から、エージェントの「視覚」機能はローカルで完結させる必要があります。

- **ローカル視覚エンジン (Local Vision Engine)**:
  - **エッジ用 VLM**: `MiniCPM-V 2.6` (約 5.5GB) 等を採用。8GB VRAM 程度の一般的な GPU で動作し、連続したスクリーンショットを用いた推論が可能です。
  - **バックエンド**: `Ollama` や `vLLM` をバックグラウンドで起動し、OpenAI Vision 互換の API を提供。
- **アクション・コントローラー (Action Controller)**:
  - **OS 制御**: `PyAutoGUI` を介して、VLM が出力した `[Click at X, Y]` 形式の座標を、実際の物理的なマウス移動やキー入力へと変換します。
  - **セーフティ・サンドボックス**: エージェントの暴走を防ぐため、初期段階では仮想マシン（VM）や特定のアプリケーション内のみに操作を制限します。

## 3. ビジネス活用シナリオ (Commercialization)

API が公開されていないレガシーなソフトウェアや、頻繁に UI が変わる SNS プラットフォームの自動化を可能にします。

### 3.1 マーケター・EC 事業者向け：UI 変更に強い SNS 運用 AI (Visual-Based Auto-Poster)

- **ターゲット**: SNS 運用代行業者、Eコマーススタジオ。
- **解決策**: 従来のクローラーは HTML 構造の変化に弱いですが、Computer Use エージェントは「見た目」で投稿ボタンを判断します。プラットフォームの仕様変更があっても、ボタンの見た目が変わらなければ自動運用を継続できます。
- **価値**: 人間と同じ操作ロジックであるため、プラットフォーム側からの Bot 検知（BAN 確率）を大幅に低減。

### 3.2 企業・エグゼクティブ向け：仮想 IT 秘書 (Desktop IT Auto-Agent)

- **ターゲット**: 非技術職の経営層、中堅企業。
- **解決策**: プロンプトの入力すら不要な「音声/テキストによる操作代行」。例：「昨日のプレゼン資料を開いて、最新の財務データをグラフ化して追加しておいて」という命令から、AI が Excel と PowerPoint を自律操作します。
- **価値**: AI をチャットボックスから解放し、「手足と目を持つ」実務代行エージェントとしてパッケージ販売。

---

## 4. 実装の第 1 歩 (Next Steps)

1.  **実戦検証 (POC)**: `PyAutoGUI` とクラウド VLM（Claude 3.5 Sonnet 等）を組み合わせ、ブラウザ操作のプロトタイプを作成。
2.  **ローカル置換**: クラウド API をローカルの `MiniCPM-V` 等に差し替え、レイテンシと座標認識の精度を検証。
3.  **ユースケースの絞り込み**: 「指定の画像を探してダウンロードする」等の単純かつ実用的な業務からテストを開始。

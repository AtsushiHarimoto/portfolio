# Moyin リーンキャンバス (Lean Canvas)

## @Overview

産業レベルの AI インタラクティブ・エンジンとしての Moyin プロジェクトのビジネスモデルを、1 枚のキャンバスに集約しました。

---

### 1-2. 課題と顧客セグメント (Problem & Customer)

| 1. 課題 (Problem)                                                                                | 2. 顧客 (Customer Segments)                                                                |
| :----------------------------------------------------------------------------------------------- | :----------------------------------------------------------------------------------------- |
| **制作コストの爆発**: 従来のアニメ制作は $50k/min を要し、個人や小規模チームでは不可能。         | **ビジュアルノベル・ゲーマー**: 深いストーリー体験を好み、IP に対して支払い意欲がある層。  |
| **制作サイクルの長期化**: Web3/TikTok の高速なトレンドに対応できない。                           | **アニメ・コンテンツ消費者**: 短尺動画を好み、隙間時間でのエンターテインメントを求める層。 |
| **競合の内容空洞化**: 既存の AI 動画はスプレッドシート的な動きに終始し、叙事詩的な深みに欠ける。 | **アーリーアダプター**: AI を活用したインディーゲーム開発に好意的な層。                    |

---

### 3-4. UVP と解決策 (UVP & Solution)

| 3. 独自の価値提案 (UVP)                                                                                   | 4. 解決策 (Solution)                                                                 |
| :-------------------------------------------------------------------------------------------------------- | :----------------------------------------------------------------------------------- |
| **Tokyo Legal (日本法準拠)**: 日本の著作権法を戦略的に活用し、クリーンかつ持続可能な AI 学習環境を構築。  | **Moyin Engine**: ComfyUI + Rust による工業化されたプロダクション・パイプライン。    |
| **Alive Character (命を宿す演出)**: 単なる画像生成ではなく、Director Agent による感情豊かな演技の自動化。 | **高効率な生産性**: 週産 2 エピソード (20分) を可能にし、コストを < $15/min に抑制。 |

---

### 5. チャネル (Channels)

- **TikTok / Shorts**: 認知獲得およびトラフィックの導線（集客）。
- **YouTube / Bilibili**: コンテンツの蓄積とファンコミュニティの形成。
- **Steam / DLsite**: 最終的なプロダクト（製品版）の販売とマネタイズ。

---

### 6-7. 収益とコスト構造 (Revenue & Cost)

| 6. 収益の流れ (Revenue Streams)                                    | 7. コスト構造 (Cost Structure)                            |
| :----------------------------------------------------------------- | :-------------------------------------------------------- |
| **広告収益**: YouTube 等（Low Margin / トラフィック重視）。        | **インフラ費**: 東京の電力コスト、AWS/Vercel、AI API 等。 |
| **ファン購読**: Patreon 等（Medium Margin / 継続性）。             | **労務費**: 開発チームの時間、特定アセットの外注等。      |
| **製品販売**: Steam ゲーム販売（High Margin / スケーラビリティ）。 | **研究開発**: 新規 LoRA 学習、ワークフローの最適化。      |

---

### 8-9. 壁壘と指標 (Moat & Metrics)

| 9. 圧倒的優位性 (Unfair Advantage)                                                              | 8. 核心指標 (Key Metrics)                                            |
| :---------------------------------------------------------------------------------------------- | :------------------------------------------------------------------- |
| **10-GPU Farm**: 自社保有の GPU 計算資源による、限界費用ゼロの試行錯誤。                        | **RPM (Revenue Per Minute)**: 1分あたりの動画収益 > $30。            |
| **Full-Stack Ownership**: コード(Rust) + モデル(LoRA) + 脚本(IP) の三位一體による模倣困難な壁。 | **WVR (Wishlist/Visits Rate)**: Steam ページからの変換率が 5% 以上。 |

# AI エージェント（Agent）と MCP の徹底解説

**本章の目標**：テック業界で頻出する「エージェント（Agent）」という言葉の本当の意味を解明します。その頭脳がどのように動作するのか、そして MCP を通じてどのように無敵のプラグインを装着できるのかを理解しましょう。

---

「AI アシスタント」や ChatGPT は皆さんご存知でしょう。しかし、なぜ今エンジニアの間で **「AI エージェント（Agent：自律エージェント）」** が話題になっているのでしょうか？
そして、環境構築時によく耳にする **MCP プロトコル（Protocol）** とは一体何なのでしょうか？

---

## 1. ChatGPT はただの事務員、エージェントは完全装備の現場部隊！

エージェントをただのチャットボット（Chatbot）だと思わないでください。両者の戦闘力の差を見てみましょう：

| 形態 | 例え話 | 戦闘力と限界 |
| --------------------------------------------- | ---------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| **通常の言語モデル<br>（例：ChatGPT Web 版）** | **「取調室に閉じ込められた事務職員」**。<br>目の前のタイプライターで机上の空論をするしかできません。 | あなた：「サーバーを再起動してくれますか？」→ 回答：「申し訳ございません。私は AI ですので、あなたのコンピュータに触れる手足がありません。」 |
| **AI エージェント<br>（例：Claude Code、Antigravity）** | **「防弾チョッキと無線機を装備した現場部隊」**。<br>公式：`頭脳（LLM）+ ツールボックス（Tools）+ 行動ロジック（ReAct）` | あなた：「プロジェクトファイルを更新して！」→ 頭脳：「了解！**ファイル読み書きツール（Tool）** を呼び出して実行します！」 |

---

## 2. エージェントの核心を解明：ReAct 思考ループ（フロントエンドの Event Loop のように）

AI にコマンドを出した後、「数分間くるくる回って」から突然大量のコードを出力する経験はありませんか？画面上で独り言を言うこともあります。
これはフリーズしているのではなく、**ReAct（Reasoning and Acting：推論と行動）** と呼ばれる内部ループを実行しているのです！

タスクに直面すると、AI エージェントの頭脳は以下の 4 ステップを無限にループします（フロントエンドの止まらない Event Loop のように）。問題が解決するまで続きます：

1. **[思考 Thought]**：「上司がウェブサイトが壊れたと言っている。まずツールを使ってエラーログ（Log）を読むべきだ。」
2. **[行動 Action]**：（ターミナルツールを呼び出し）→ `grep で error を検索するコマンドを実行！`
3. **[観察 Observation]**：（ツールがエラーメッセージを頭脳に返す）：「ログにデータベースパスワードエラーと書いてある。」
4. **[最終回答 Final Answer]**：「上司、パスワードを修正しました。ウェブサイトは復旧しました！」

> **ヒント**：
> 会話ログに `<thought>` タグが表示されることがありますが、それは ReAct ループの内部独白です。
> 良い上司のコツは：**考えさせてあげましょう。考え終われば成果物を出してくれます。**

---

## 3. MCP（Model Context Protocol）はどれほど強力なのか？

先ほど、AI にさまざまなツールを装備させることを説明しました。
しかし、以前のエンジニアは大きな災難に直面していました。マイクロソフトが作ったツールは Google のモデルでは認識されず、Google が作ったツールは Claude モデルでは認識されない。インストールだけで疲弊していました（`npm` が登場する前の暗黒時代のように）。

そこで、Claude モデルを開発した Anthropic 社が「全世界の AI に恩恵をもたらす」と称される通信標準を発表しました：**MCP（Model Context Protocol：モデルコンテキストプロトコル）**。

> **例え話：MCP は AI 界の万能 USB Type-C インターフェースです！（またはパッケージ管理の npm install）**

どのモデルであろうと、全員がこの標準 Type-C インターフェースを装備していれば、
エンジニアは「Google Stitch デザイン図スキャナープラグイン（MCP Server）」を一度開発するだけで、誰でも接続してすぐに使えるようになります！

MCP があれば：

- 接続するだけで、AI がデザインファイル（Google Stitch）を読めるようになります。
- 接続するだけで、AI が Slack を開いてグループメッセージを監視できるようになります。
- 接続するだけで、AI がローカルの Google ドライブに直接アクセスできるようになります。

## 検収チェックリスト

チャットパネルをもはや昔のおしゃべりだけの ChatGPT だと思わないでください。今あなたの前にいるのは、手足を持つ完全装備の現場エージェントです。

- [ ] AI エージェントは単なる頭脳ではなく、ツールを装備し自ら解決策を見つけられる自律システムであることを理解しました。
- [ ] AI が返答前にくるくる回る理由を理解しました（ReAct ループ：思考 → 行動 → 観察の Event Loop を実行しているため）。
- [ ] MCP が万能 USB Type-C インターフェースのようなもので、MCP に対応した強力なツールなら何でも AI に接続してすぐに使えることを理解しました。

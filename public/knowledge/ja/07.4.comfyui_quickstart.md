# ComfyUI クイックスタート：ノードベースの自動化画像制作パイプライン

## @Overview

AI による画像生成を「工業化」するためには、場当たり的なプロンプト入力から脱却し、精密に制御された「生産ライン」を構築する必要があります。本稿では、Moyind プロジェクトでも採用されている、JSON スクリプトによる完全自動化画像制作が可能な「ComfyUI」の導入と運用について解説します。

---

## 1. 入門：ブラックボックスを解体する「ノード哲学」

ComfyUI の最大の特徴は、画像生成の全工程を「ノード (Nodes)」として可視化し、それらを「リンク (Links)」で繋ぐことで独自のアルゴリズムを組み立てられる点にあります。

- **Node (演算ノード)**: モデルのロード、プロンプトの解析、サンプリング（描画）など、特定の機能を担う「作業員」です。
- **Link (データケーブル)**: ノード間でベクトルデータや潜在空間の情報を転送する「電線」です。
- **Workflow (ワークフロー)**: これら全体の構成図を JSON 形式で保存した「自動化の台本」です。一度作成すれば、再利用や他者との共有が容易になります。

---

## 2. 環境構築：システムを汚さない「ポータブル運用」

AI 関連ソフトウェアの導入で最も懸念されるのは、システム環境（Python パスなど）の衝突です。

- **推奨**: **Portable (免インストール版)** の利用を強く推奨します。
- **ディレクトリ構造**: `models/` に Checkpoint を、`custom_nodes/` に拡張プラグインを配置するだけで、完全に独立した「サンドボックス環境」として動作します。
- **低 VRAM 対策**: 搭載 GPU のメモリが 6GB 未満の場合は、起動引数に `--lowvram` を追加することで、低スペック環境でも最小限の動作を確保できます。

---

## 3. 必須ツール：ComfyUI Manager (管理自動化)

ComfyUI を使い始めて最初に行うべきは「ComfyUI Manager」の導入です。

- **依存関係の自動解決**: 外部から入手したワークフロー JSON を開いた際、不足しているカスタムノード（プラグイン）をスキャンし、一括でインストールしてくれる不可欠な管理ツールです。

---

## 4. プログラマティック生成：API 駆動による外部連携

ComfyUI を Moyind エンジンの「描画サーバー」として機能させるための高度な手法です。

1.  **Dev Mode の有効化**: 設定から「Enable Dev mode Options」を ON にし、**「Save (API Format)」** を選択することで、UI 情報を省いた「純粋な論理骨格」としての JSON データを取得できます。
2.  **外部からの命令実行**: 取得した API 用 JSON を Python スクリプト等から HTTP リクエスト (POST) で送信することで、バックグラウンドでの無人画像生成が可能になります。これにより、ゲーム内の進展に合わせてリアルタイムに画像を生成し、プレイヤーに届けるといった動的な演出が実現します。

---

## 💡 実務への応用アドバイス

自動化パイプラインを安定稼働させるためには、以下の運用を検討してください：

- **キュー (Queue) 管理**: 大量の生成リクエストを捌く際は、サーバー側でのリクエスト・キューイングを実装し、VRAM の溢れ (OOM) を防止してください。
- **パラメータの動的オーバーライド**: 基本となるワークフロー JSON をテンプレートとし、リクエストごとにプロンプトや LoRA の強度を動的に書き換えることで、バリエーション豊かなアウトプットを効率的に生成できます。

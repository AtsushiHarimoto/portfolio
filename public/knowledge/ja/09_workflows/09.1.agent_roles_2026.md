# AI エージェント分担規約（2026年版）：Claude, Codex, Antigravity の三位一体協調 (Agent Roles 2026)

## @Overview

AI エージェントによる開発が当たり前となった 2026年において、単一の AI にすべてを任せるのは非効率です。Moyin プロジェクトでは、職能の異なる 3つの AI エージェント（Claude, Codex, Antigravity）を適材適所で運用し、人間は「意思決定者」として君臨する体制を構築しています。

本ドキュメントでは、各エージェントの役割、標準アウトプット、および連携マトリックスについて定義します。

---

## 1. ロール定義（三位一体の役割分担）

### 🟢 Claude — アーキテクト / 戦略層

全体の設計図を描き、技術的な意思決定を下す「ブレーン」です。

- **職務**: 業務要件の分析、システムアーキテクチャ設計、大規模リファクタリングの戦略立案、コードレビュー。
- **強み**: 抽象的な思考、論理の整合性チェック、リスク予測。
- **標準成果物**: 構造化された実装設計書、コードレビュー報告書、TODO マトリックス。

### 🔵 Codex — バックエンド / データ層

サーバーサイドのロジックとデータの流れを完璧に制御する「心臓部」です。

- **職務**: API エンドポイント開発、データベース設計・移行、パフォーマンスチューニング、非同期処理の最適化。
- **強み**: 正確なコード生成、複雑な SQL 操作、データのクレンジング。
- **標準成果物**: 堅牢なサーバーサイドコード、マイグレーションスクリプト、ユニットテスト群。

### 🔴 Antigravity — UI・UX / フロントエンド層

ユーザーが直接触れる体験を美しく、滑らかに構築する「顔」です。

- **職務**: デザイン案の忠実なコーディング、状態管理（State）の実装、共通 UI コンポーネントの開発、プロトタイピング。
- **強み**: CSS/アニメーションの微調整、直感的な React/Vue コンポーネント設計。
- **標準成果物**: 洗練されたフロントエンドソース、UI コンポーネントライブラリ、描画検証ログ。

---

## 2. 連携マトリックス（どのように協力するか）

### 模式 A：トップダウン設計（Claude ➝ 実行層）

- **フロー**: Claude が設計と TODO を作成（`workspace/plans/`）➝ Codex または Antigravity がそれを受け取り、具体的なコーディングを実行。
- **用途**: 新機能開発、大規模なソースのリノベーション、複雑なマルチシステム連携。

### 模式 B：フラットな直接実行（User ➝ 各エージェント）

- **フロー**: ユーザーが明確な指示を出し、特定のエージェント（Codex または Antigravity）が即座に変更を適用。
- **用途**: バグ修正 (Hotfix)、文言の修正、小規模な定形タスク。

### 模式 C：ガードレール駆動（実行層 ➝ Claude 審査）

- **フロー**: 他のエージェントが作成したコードを、Claude が客観的な視点でレビューし、品質基準をクリアするまで修正を命じる。
- **用途**: プロダクション環境へのデプロイ直前、セキュリティチェック、コードの美しさの追求。

---

## 3. 2026年の鉄の掟

- **謀略 (PLAN) 先行**: テンプレートにない複雑なタスクは、まず Claude に計画を立てさせること。脊髄反射でのコーディングは禁止。
- **エビデンス主義**: 「直しました」という言葉だけでは不十分。必ずコマンドの出力結果、テストログ、またはビジュアルな証拠（スクリーンショット等）を提出すること。
- **人間の最終承認**: 人間（ユーザー）の明確な `OK` がない限り、いかなるファイルの保存や削除も実行してはならない。

この階層化されたエージェント体制こそが、複雑化するソフトウェア開発を「光速」で、かつ「安全」に進めるための Moyin 流ソリューションです。

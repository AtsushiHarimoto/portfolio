# AI 協作指令速查表 (Cheat Sheet)

> 忘記指令？運行專案根目錄下的 `./help.ps1` (Win) 或 `./help.sh` (Mac/Linux) 即可查看互動式菜單。

---

## 🚀 核心開發

| 任務           | 指令 (Prompt) | 說明                                            |
| -------------- | ------------- | ----------------------------------------------- |
| **開發新功能** | `/run-dev`    | 標準開發 SOP：文檔檢查 → 計劃 → 執行 → 驗證     |
| **排查 Bug**   | `/run-bg`     | 系統化 Bug 診斷：收集證據 → 根因分析 → 修復方案 |
| **執行測試**   | `/run-qa`     | 測試執行 SOP：制定計劃 → 執行測試 → 生成報告    |

## 📦 版本控制與歸檔

| 任務         | 指令 (Prompt)        | 說明                                                |
| ------------ | -------------------- | --------------------------------------------------- |
| **提交代碼** | `/run-archive`       | **推薦**。自動分析 diff，生成 `1.repo.md`，提交 git   |
| **更新日誌** | `/run-update-blocks` | 掃描今日 commits，自動更新 `workspace/work/blocks/` |

## 📚 知識管理與文檔

| 任務          | 指令 (Prompt)       | 說明                                                 |
| ------------- | ------------------- | ---------------------------------------------------- | --- |
| **同步文檔**  | `/run-sync-docs`    | 全量備份文檔到 `output/` (給 GPT Projects 用)        |
| **Wiki 管理** | `/run-wiki`         | 支援子命令：`inbox`, `sync`, `conversation`, `index` |     |
| **日語同步**  | `/deliverables_jap` | 將繁體中文文檔同步為日語交付物                       |

## 🤖 Claude Code 專用 (VS Code)

直接在 Claude 對話框輸入：

| 範圍     | 指令範例                                          |
| -------- | ------------------------------------------------- |
| **GAME** | `GAME - 請為 src/vn/runtime/ 補全繁體中文註釋...` |
| **WEB**  | `WEB - 請重構 WorldBuilder 的狀態管理...`         |
| **API**  | `API - 請解釋 web2api 的路由邏輯...`              |
| **TEST** | `TEST - 請為新功能編寫 E2E 測試...`               |

> Claude Code 會自動識別簡稱並定位正確的工作目錄。

---

## 💡 使用技巧

1. **永遠先 PLAN**：不要讓 AI 直接改代碼，先要求 `/run-dev` 或手動輸入 `PLAN:`。
2. **小步快跑**：一個 Prompt 只做一件事（如：只改一個文件，或只修一個 Bug）。
3. **證據優先**：報 Bug 時，先用 `/run-bg` 並提供日誌/截圖。

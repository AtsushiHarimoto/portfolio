# タスク管理システムの設計と実装

> **種類**: ワークフロー (Workflow) / プロジェクト管理
> **日付**: 2026-01-11
> **出典**: 会話 - タスク管理システム構築

---

## 概要

JIRA に類似した軽量タスク管理システムを構築し、すべての開発タスク、システムバグ、機能要望を統一的に管理します。過去の Scope 分散と入口混乱の課題を解決します。

---

## 背景

### 歴史的課題

1. **Scope の分散管理**：各サブシステムが個別に `docs/scope/` 内で管理し、グローバルな統一ビューが欠如。
2. **タスク入口の混乱**：`workspace/ideas` に雑多なアイテムが堆積、チャットからの臨時タスク依頼、標準化された受付・配分拠点の欠如。
3. **追跡メカニズムの欠如**：シームレスな TODO リストビューがない、実行中/完了/アーカイブ済みの把握が困難、履歴タスクの迅速な検索が困難。

### 設計目標

- シングルポイントオブエントリー (Single Point of Entry) で全タスクを統括。
- 最小限のステート遷移フローを実現。
- 運用コストを最小化しつつグローバル検索を可能に。
- 既存の Scope 管理システムとシームレスに統合。

## 詳細設計

### ディレクトリ構造

```
workspace/
├── issues/                      # 統一タスク管理ルート
│   ├── README.md               # システム利用ガイド
│   ├── INBOX/                  # 新規タスク受信ボックス（未割当）
│   ├── TODO/                   # バックログ（計画済み、未着手）
│   ├── DOING/                  # 進行中
│   ├── DONE/                   # 完了（アーカイブ待ち）
│   └── ARCHIVE/                # 履歴アーカイブ（月別）
│       └── {YYYY-MM}/
├── dashboard/                   # タスク統計ダッシュボード
│   └── README.md
└── progress/                    # 大規模タスク進捗追跡
    └── {TASK_ID}_1.progress.md
```

### 統一命名規則

```
ISSUE-{SEQ}-{SCOPE}-{TYPE}-{SHORT_DESC}

パラメータ:
- SEQ: 連番 (001, 002...)
- SCOPE: API | WEB | GAME | TEST | META
- TYPE: BUG | FEAT | DOC | REFACTOR | TASK
- SHORT_DESC: 2-3 単語の簡潔な説明（英語、snake_case）

例:
- ISSUE-042-GAME-BUG-save_load_fail
- ISSUE-043-WEB-FEAT-editor_ui
- ISSUE-044-META-DOC-workflow_update
```

### ステート遷移マトリクス

```
INBOX -> TODO -> DOING -> DONE -> ARCHIVE/{月}/
```

### 配下ワークフロー

1. **`/new_issue`**：新規イシュー作成 — インタラクティブ Q&A で要件収集、自動採番、標準ディレクトリと README 生成。
2. **`/move_issue`**：ステート遷移 — あいまい ID マッチング、README とステート履歴の自動更新、ガード付き。
3. **`/list_issues`**：イシュー検索 — 多次元フィルタリング（Status / Scope / Type / Priority）、サマリー/詳細の 2 ビュー。
4. **`/archive`**：深度アーカイブ — 3 モード（Issues Only / Code Only / Full Archive）、関連チェック付き。

## 適用シナリオ

- **日常開発イテレーション**：開発ライフサイクルの End-to-End 追跡。
- **バグ管理**：報告から解決までの全ライフサイクル管理。
- **機能拡張**：アイデアから仕様収束、コード実装までの全行程管理。
- **マイルストーンプロジェクト管理**：フェーズごとのタスク配分と進捗監視。

---

## 注意事項

- **命名厳格規範**：`SHORT_DESC` は英語 snake_case 必須。
- **ステートスキップ禁止**：INBOX から DONE への直接ジャンプは非推奨。
- **定期アーカイブ**：月末に `/archive` を実行してシステムを軽量に維持。
- **README 完全記入**：各 ISSUE の README に情報の欠落がないよう確保。

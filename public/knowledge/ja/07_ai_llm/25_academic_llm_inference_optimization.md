## 1. 空間で時間を買う王道：KV Cache (鍵・値キャッシュ)

以前お話しした通り、大規模言語モデル（LLM）は「しりとり・接龍」のように文字を出力します。新しい一文字を出すたびに、それまでの全テキストを「最初から計算し直して」次の一文字を決定します。
例えば 1000文字目を出力するとき、前の 999文字の行列演算を繰り返すのは、あまりにも非効率です。これは $\mathcal{O}(N^2)$ の無駄な重労働です。

### 💡 科学者の「手抜き」妙案

前の 999文字の `Key` と `Value` はすでに一度計算されているはずです。なら、それを「冷蔵（キャッシュ）」しておけばいい。

- GPU メモリ内に専用の領域を確保し、計算済みの「部品」を保存します。
- 新しい一文字が入ってきたとき、その文字の `Query` だけを計算し、倉庫（KV Cache）から旧部品を取り出して掛け合わせます。
- この手法により、数千キロの重複した道のりをショートカットできます。現代の LLM サーバーにおいて必須の装備です。

---

## 2. ハードウェアの境界を突破：FlashAttention (閃電注意術)

KV Cache があっても、依然としてグラボの物理的な壁に直面します。それが **メモリの壁 (Memory Wall)** です。GPU の計算コア自体は光速で動きますが、データをメインメモリから計算コアへ運ぶ速度は、牛歩のように遅いのです。GPU は一生のうち 80% の時間を「データの到着待ち」で暇を持て余しています。

### 🚀 FlashAttention による救済

スタンフォード大学の研究者が考案した神レベルの最適化が **タイリング (Tiling)** です。

- 従来は巨大なデータの塊を丸ごとメインメモリとコアの間で往復させていました。
- FlashAttention は、巨大なアテンション行列を「寿司」のように小さく切り分けます。
- **重要なポイント**: GPU コアがその小さな断片を読み込んだ際、**超高速な内部キャッシュ (SRAM) 内で一気に計算を完結させます。** 途中でデータを外に出さない（書き戻さない）のです。
- この技術により、メモリの複雑度が劇的に低下しました。これなしでは、100万トークンを読みこなす Claude 3 のような怪物は誕生し得ませんでした。

## 3. デヴィッドとゴリアテ：投機的デコード (Speculative Decoding)

これは、現在最も「巧妙」な手法です。
もし、わずか 10億パラメータの「少し非力な小兵（1Bモデル）」と、1750億パラメータの「絶対的権威（GPT-4級の大神）」を組ませたらどうなるでしょうか？

### 🎲 「先斬後奏」の賭け

1.  **下書きの暴走**: 足の速い「小兵モデル」が、一瞬でそれに続く 5文字を適当に予測します（例：「リンゴ、パイ、は、甘、い」）。
2.  **大神による審判**: その 5文字をまとめて「大神」に渡します。大神はゼロから考えるのではなく、小兵の答えが正しいか「○×チェック（検証）」するだけです。これは並列処理が可能です。
3.  大神の判定：「最初の 4文字は正解。最後の一文字は『美味しい』に変えたほうがいい」。
4.  **結果**: 大神が 1回計算する時間で、5文字分の高品質なテキストが生成されました。

この「小兵が予測し、大将が検証する」投機的戦略は、文字出力の遅延を劇的に短縮します。次世代 AI プロダクトの標準装備となるでしょう。

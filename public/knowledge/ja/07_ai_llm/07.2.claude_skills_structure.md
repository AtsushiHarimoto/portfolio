# AI アシスタント最強プラグイン：Claude Code 技能拡張（Skills）ディレクトリ軍事規格 (Claude Skills Structure)

## @Overview

こんにちは、AKIRA です。
今回は、AI アシスタント（Claude Code や Antigravity 等）に、世界を揺るがすような「カスタムスキル」を実装する方法についてお話しします。

AI の能力を拡張するのを、単に指示が書かれた Word や Markdown ファイルを読ませるだけだと思っていませんか？もしそうなら、このエンジニアリング体系を甘く見すぎています。AI は「環境センシング」を非常に重視します。ディレクトリを適切に切り分け、ファイル名を正確に記述しなければ、どれだけ叫んでも AI は最強の外掛指令を認識してくれません。

Moyin プロジェクトにおける「外部スキルパック (Skills)」のディレクトリ構造、その軍事規格並みに厳格な設定基準について解説します。

---

## 1. 致命的なビギナーの誤解：単一ファイル集中型の罠

経験の浅い開発者はこう考えがちです。「5つのスキルがあるなら、1つのファイルにまとめて書けばいいじゃないか」と。彼らは `.claude/skills/everything.md` を作成し、そこにすべてのスクリプトを詰め込みます。
**断言しますが、これは現代の AI Agent スキャンメカニズムにおいて、完全かつ根本的な間違いです。**

- **鉄則**: `@スキル名` による正確なトリガーをアンロックするためには、**個々の独立したスキル（Skill）が、それぞれ専用の物理ディレクトリを所有していなければなりません。**
- **同期原則**: メイン開発リポジトリは `.agent/skills/` に置きます。Claude Code ユーザー向けには、スクリプト一発で `.claude/skills/` 以下に同期される仕組みになっています。

---

## 2. 標準セキュリティ級ディレクトリ構造 (Skills Layout)

AI がスキャンした瞬間に「スキルの鍵」をキャッシュへ保存できるよう、以下の階層を厳守してください。

```text
.agent/skills/          # (プロジェクトのスキル本拠地)
├── handoff/            # [各スキルが独立した城を持つ]
│   ├── SKILL.md        # ⬅️ 絶対的な核心！スキルの主説明書
│   └── TEMPLATE.md     # 補助用の各種テンプレート、参照ファイル
│
├── new_issue/          # 「@new_issue」入力時のレスポンス定義
│   └── SKILL.md
│
└── dev/                # 「@dev」入力時のレスポンス定義
    └── SKILL.md
```

### なぜスキルごとにフォルダを作る必要があるのか？

高度なスキル（例えば「プロジェクト引き継ぎ `handoff`」）は、単なるコマンドではなく、出力用の JSON 規格や Markdown テンプレートを必要とすることが多いためです。
**ディレクトリとしてパッケージ化することで、AI は `@handoff` をロードした瞬間に、フォルダ内の「フルセット」の背景情報を一括で読み込むことができます。** このカプセル化こそが、エンタープライズ級開発の底力です。

---

## 3. `SKILL.md`：成否を決める「身分証」

各スキルディレクトリ内には、正確に命名された `SKILL.md` が必須です。このファイルの冒頭にある、パスワードのような **YAML Frontmatter** ブロックが最も重要です。ここを書き漏らすと、AI はそのファイルをゴミとして無視します。

```markdown
---
name: new_issue
description: "新しいタスク追跡エンティティの作成を支援。トリガー：@new_issue"
---

# 新規タスク作成 (new_issue)

（ここに具体的な SOP 手順の内容が続く...）
```

### ここでチェックすべき 2 つのポイント：

1.  **`name`**: ディレクトリ名と完全に一致させること！これが AI にとっての「呼び出し合言葉」になります。
2.  **`description`**: 一文で簡潔に記述すること。そして重要——**末尾に必ず「トリガー：@xxx」と明記すること！** これはセマンティック・ウェイトを高めるための重要なラベルです。これがないと、AI の怠慢なスキャンエンジンがこの必殺技を見落とす可能性があります。

## 💡 Vibecoding 成功の秘訣：AI への発注

AI にプロジェクトをまるで自分の家のように管理させるための最高奥義です。新しいスキルパックを AI に書かせる際は、出資者の強気な態度でこう命令してください。

> 🗣️ `「AI 助教！今から新しい @commit スキルプラグインを書いてもらう。
この軍事規格プロセスを叩き込め！第一に、.agent/skills/ の下に commit という名前の物理ディレクトリを作成しろ。
第二に、その中に SKILL.md を生成し、冒頭にディレクトリ名と一致する YAML Frontmatter を含め、
description にトリガー方法を明記しろ。
第三に、無駄な教本は書くな。Git Staging Area をスキャンして Conventional Commit メッセージを
自動生成する硬派な SOP を直接実装しろ！
完了したら同期スクリプトを走らせ、この新スキルを有効化しろ。さあ、かかれ！」`

ディレクトリと Frontmatter という 2 つの門をしっかり閉めれば、AI という頭脳はあなたの手足となって動いてくれます。さあ、最初の `@` スキルを試してみましょう！

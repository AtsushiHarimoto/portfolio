## 1. オフラインモデルの中枢：Ollama デプロイ実戦

もしあなたが NVIDIA RTX シリーズのような優れたグラフィックボードを持っているなら、OpenAI の混雑した道に並ぶのはやめましょう。**Ollama** をインストールしてください。これ一つで、様々な神レベルのオープンソースモデルをローカルで一クリック起動できます。

### 📦 VRAM（ビデオメモリ）に合わせて「嫁（モデル）」を選ぶ

あなたのグラボに何 GB のメモリがあるかで、動かせる AI の賢さが決まります。

- **2-4GB (エントリー機)**: `llama3.2:3b` 一択です。少しおバカさんですが、API 形式のテストや簡単な NPC 対話には十分です。
- **6-8GB (主流スペック / RTX 3060等)**: **黄金の標準構成です！** 迷わず `qwen2.5:7b` を選んでください。アリババの Qwen は中国語（および日本語）の語感が非常に鋭く、現在の Galgame エンジンの第一候補です。
- **12GB (ハイエンド)**: `qwen2.5:14b` が動かせます。推論の深さが一段階上がります。
- **24GB (RTX 3090/4090)**: おめでとうございます。`32b` クラスや `qwq` プレビュー版の圧倒的な知能を体験できます。記憶力と論理的思考力は「無敵」に近いレベルです。

## 2. クラウド認証の黒魔術：Firefox Cookie 奪取スキーム

最強のクラウドモデル（Gemini 1.5 Pro など）を使いたいけれど、公式 API の制限が厳しかったり、ウェブ版の無料枠を使いたい場合があります。しかし、Google や Claude の Cookie は強力に暗号化されており、通常の Python スクリプトでは盗み出せません。

### 🦊 なぜ Firefox を選ぶのか？

Chrome は Cookie をシステムのハードウェアキーにロックしているため、復号しようとするとウイルス対策ソフトが激しく反応します。一方、**Firefox** は比較的義理堅い。Cookie は SQLite データベースに保存されています。

**Moyin のワークフロー：**

1. スクリプトが自動的に Firefox を起動。
2. あなたが手動でウェブ版にログイン。
3. `cookie_check.py` がバックグラウンドで Firefox のローカル保存領域に「瞬間移動」し、認証トークンを無傷で抽出して API Gateway へ渡します。
   この手法の成功率はほぼ 100% で、驚くほど安定しています。

## 3. ミリ秒単位のレスポンス：Native Context 管理術

ローカルの Ollama を使うなら、OpenAI 互換レイヤーのような「余計な中継ぎ」は使わないでください。ビジュアルノベル特有の「クリックした瞬間に文字が出る」快感を追求するため、私たちは Ollama 本来の `/api/chat` プロトコルを直接叩きます。

- **KV Cache の再利用**: 会話が終わるたびに、Ollama は `context` という「記憶のスナップショット」を返します。
- **次元削減攻撃**: 次の会話では、過去の履歴をすべて送り直す必要はありません。その「スナップショット」に最新の一文を加えるだけです。
- これだけで、本来 3 秒かかっていた推論が 0.5 秒以下に短縮されます。これがプロのエンジニアのこだわりです。

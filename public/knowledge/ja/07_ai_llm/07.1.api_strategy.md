# AI Galgame エンジン：API コストを「利益」に変える、サバイバル・マネタイズ戦略 (API Strategy)

## @Overview

こんにちは、AKIRA です。
今回は、インディーゲームスタジオが来月も生き残れるかどうかを左右する、非常に現実的で残酷な問題についてお話しします。それは、**API の請求額とビジネス的なマネタイズ**です。

現在、AI 駆動のインタラクティブゲームや AI 彼女、あるいは謎解き Galgame が流行っています。しかし、何も考えずに OpenAI を直接叩き続けていれば、ゲームがリリースされる前に、数百万トークンの消費によってクレジットカードの限度額が吹き飛ぶことは間違いありません。

Moyin プロジェクトが、究極の知性を追求しながら、いかにして API コストを極限まで抑え込み、利益へと繋げているのか。その戦略を解説します。

---

## 1. なぜお金が溶けるのか？トークン爆発の 2 大原因

インタラクティブ・ストーリーテリングにおいて、初心者が陥りやすい罠が 2 つあります。

1.  **💸 ステートレスの「忘却コスト」**:
    公式 API（GPT-4o 等）は、実はひどい「物忘れ」をします。NPC に「彼女はあなたの幼馴染である」という設定を覚えさせ続けるためには、会話のたびに数千、下手をすれば数万トークンの世界観設定と履歴を「毎回すべて」送り直す必要があります。プレイ時間が長引くほど、インプットトークンの増加速度はロケットのように跳ね上がります。
2.  **🚨 リバース API の致命的なリスク**:
    コストを削るために、Web 版などを勝手に解析（Reverse）した無料 API に手を出すのは、命懸けのギャンブルです。これらは極めて不安定で、プラットフォームがセキュリティを強化した瞬間に `403 Forbidden` を吐いてゲームが沈黙します。商業ベースで運営するなら、この手の不安定な「黒魔術」には絶対に手を出してはいけません。

---

## 2. 強者が実践する「3 段階エンジン進化論」

「知性」「遅延」「財布」という不可能の三角形の中で生き残るため、Moyin エンジンは以下の 3 つの進化段階を採用しています。

### Phase 0: 状態マシンと「冷酷なローリング・サマリー」

履歴をすべて AI に投げるのはやめましょう。エンジン内に精密な `state.json` を保持し、今この瞬間に必要な変数だけを AI に渡します。さらに、直近 8 ターンの会話以外は即座に「要約コマンド」にかけ、1 行の「思い出サマリー」に圧縮します。これにより、トークンコストを常に一定の低い範囲に固定（定数化）し、コスト爆発を根絶します。

### Phase 1: 脳の分離（コントローラーとライターのデカップリング）

安価な「小脳 (Controller)」にプレイヤーの雑談をフィルタリングさせ、重要なフラグが立った時や高難度の謎解きが発生した時にだけ、非常に賢く高価な「大脳 (Writer - Claude 3.5 等)」を呼び出して神がかった脚本を書かせます。この「漏斗型インターセプト」により、不要なコストを 80% 以上削減可能です。

### Phase 2: 究極形態「ローカル推論 (Localized Logic)」

私たちの終着点は、**インターネットを介さず、プレイヤーの PC でモデルを回すこと**です。`llama.cpp` と量子化の魔術を使い、Llama-3 や Qwen といった 8B 級の高性能モデルを RTX シリーズのグラフィックボードへ直接ロードします。

- **メリット**: API コストは実質「ゼロ」。プライバシーは完璧（データがサーバーへ飛ばない）。そして遅延ゼロの爆速レスポンス。

---

## 3. この AI エンジンをいかにマネタイズするか？

1.  **初期：ライセンスの借用 (BYOK - Bring Your Own Key)**:
    プレイヤーが自分の API Key をゲームに入力する形式です。私たちは何を発具しているのか？それは**「精緻なプロンプト・フレームワーク」と「記憶管理アルゴリズム」**です。知能そのものはコモディティ化しますが、その「魂の構造」こそが価値になります。
2.  **中期：特定モデルのパッケージ販売 (DLC)**:
    特定のキャラクターに特化して微調整（Fine-tune）したローカル用モデルパックを販売します。
3.  **長期：ハードウェアを含むソリューション提供**:
    ハイエンド PC を持つユーザー向けに、究極のネイティブ推論体験を付加価値として提供します。

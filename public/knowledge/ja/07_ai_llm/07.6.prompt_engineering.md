# AI を支配する権能：プロンプト・エンジニアリング (Prompt Engineering) 実戦術

## @Overview

こんにちは、AKIRA です。
今回は、ここ数年でもてはやされている **プロンプト・エンジニアリング (Prompt Engineering)** についてお話しします。

多くの人は、この学問を「文学的創作」だと思い込み、感傷的で情緒的な小説のプロットをひねり出すことに心血を注いでいます。
**断言しますが、本物の AI ソフトウェアを開発したいなら、プロンプトは文学ではなく「エンジニアリング・コントロール（工程制御）」です。**

AI（大言語モデル）の本質は、演技力は抜群だが常識が全くない「即興劇の役者」のようなものです。正確な台本の規範を与えなければ、その出力は収集がつかないほど発散し、最悪の場合、あなたのバックエンド・システムを破壊します。Moyin プロジェクトが、いかにしてこの荒馬のような AI を「絶対的な統制力」で飼いならしているのか、その手法を解体します。

## 1. プロンプト二軌制：真の演出家は誰か？

私たちの API プロトコルでは、命令を 2 つの全く異なる性質に切り分けて管理しています。これらを混同してはいけません。

### 👑 System Prompt (システム・プロンプト)

これは「演出家の見えない拡声器」です。プログラムのバックエンドに深く埋め込まれており、一般のユーザーからは見えません。

- **権威性**: 最も高い優先度を持ちます。ユーザーが外からどれほど脅迫や誘惑を行っても、AI は心の底でこの防衛線を死守しなければなりません。
- **任務**: NPC の人格を固定します（例：「あなたはクールな性格の○○です。句読点は全角のみ使用してください」「返答は JSON 形式のみ。余計な解説は一切禁止」）。

### 🗣️ User Prompt (ユーザー・プロンプト)

これは「舞台上でリアルタイムに交わされる台詞」です。

- **任務**: 刻一刻と変わるインタラクティブな状態を伝えます（例：「プレイヤーがあなたの頭を撫でた」、あるいはシステムが挿入する環境パラメータ：「【現在状況】夜の10時、月明かりが乏しい」）。

## 2. 出力形式の「絶対命令」：JSON 強制拘束

これが最も重要な点です。バックエンドのプログラムにとって、AI が書いた台本がどれほど美しくても意味がありません。もし返答の冒頭に「承知しました。こちらがプロットです：」という一言が添えられていれば、**あなたの JSON パーサーは即座にクラッシュし、エラーを吐きます。**

### 🛡️ 実戦的な防衛 3 ステップ：

1.  **一撃必殺の範本 (One-Shot Example)**: 言葉で説明するだけでなく、System Prompt の直後に「完璧な出力 JSON 形式」の例を叩き込みます。「一文字も増やすな、括弧一つも減らすな」と命じます。
2.  **思考プロセスの隠蔽 (Hidden CoT)**: AI に思考（例：相手の罠を見抜くかどうかの判断）をさせる必要がある場合、`<thought>` タグ内で考えさせます。ただし、API ゲートウェイで正規表現を用いてこの「思考のノイズ」をすべてフィルタリングし、純粋な JSON だけをフロントエンドへ送ります。
3.  **自己修復メカニズム (Self-Healing)**: 万が一、AI が括弧を閉じ忘れたりした場合はどうするか？私たちの Gateway はその例外をキャッチし、エラー内容を AI に突き返します。「お前の出力は壊れている！4行目にカンマが足りない。今すぐ直して送り直せ！」

## 3. 踏んでからでは遅い：フォーマット崩壊の地雷原

プログラムと連携する際、以下の地雷は全力で回避しなければなりません。

- **Markdown 接頭辞汚染**: AI は JSON の前後に \`\````json\`\`\` タグを付けたがります。見た目は綺麗ですが、プログラムにとっては致命的なエラーです。プロンプトで厳禁するか、バックエンドで強制的に `replace` して削除します。
- **ダブルクォート地獄 (Quote Hell)**: JSON の文法は半角ダブルクォートで括られます。もし劇中の台詞も `"食べた？"` と記述されると、引用符が衝突して JSON 構造が壊れます。
  - **解決策**: セリフを書く際は、**一律で全角の鍵括弧「 」** を使うよう強制します。これにより、技術層と内容層を完全に絶縁できます。

## 💡 Vibecoding 成功の秘訣：AI への発注

AI に一切の選択肢を与えず、忠実な兵士として従わせるための、大魔王のような気迫を持った命令です。

> 🗣️ `「AI 助教！よく聞け。今から NPC 対話インターフェースの System Prompt を設計する。
第一に、【JSON 出力モード】を徹底しろ。解説などの余計な一言が含まれていれば解析失敗と見なし、即座にリトライさせる。
第二に、【One-Shot 範本】を導入しろ。期待するナラティブ内容、state_delta、NPC の心理状態をすべてフォーマット化してテンプレートに記述しろ。
第三に、JSON 構造の混乱を防ぐため、セリフは必ず全角の【「 」】で囲むことを宣言しろ。
最後に、解析クラッシュ時の自己修復（Retry）ロジックを設計し、安定度 100% のプロンプト・アーキテクチャを生成しろ。さあ、かかれ！」`

優れたプロンプト・エンジニアリングとは、AI に「選択の余地はない」と思わせ、あなたの命令どおりに動くよう管理することなのです。

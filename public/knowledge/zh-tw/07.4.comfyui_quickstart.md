# 07.4. 自動化影像生產線：ComfyUI 節點式流水線實戰指南

## @概覽

要將 AI 生成技術實現「工業化」，必須脫離傳統「手動輸入 Prompt、看運氣出圖」的隨機階段。在 Moyind 專案中，**ComfyUI** 並非單純的繪圖工具，它是一條精密且完全可程式化控制的「自動化生產流水線」。透過 JSON 腳本，我們能一鍵實現影音資產的大量產出。

---

## 1. 核心哲學：拆解黑箱的「節點運算」

ComfyUI 的核心在於將複雜的生成過程拆解為「節點 (Nodes)」與「連線 (Links)」。

- **Node (運算節點)**: 扮演不同的功能角色。一個負責載入模型、一個負責編碼 Prompt、另一個負責執行擴散採樣。
- **Link (資料電纜)**: 在節點間傳輸張量 (Tensor) 數據。這讓開發者能像搭積木一樣，精準控制生成過程中的每一個中間步驟。
- **Workflow (工作流)**: 所有的連線結構可存儲為 `.json` 檔案。這就是我們的「自動化劇本」，能跨平台複現並受版本管理。

---

## 2. 環境建置：穩定至上的「免安裝」策略

在開發環境中跑重型 AI 軟體，最怕環境衝突。

- **推薦做法**: 一律使用 **Portable (可攜版)** 建立沙盒環境。
- **目錄規範**: 模型放置於 `models/`，插件放置於 `custom_nodes/`。其內建的 `python_embedded` 能確保 AI 庫不會干擾系統全局路徑。
- 👉 **低顯存方案**: 若 VRAM 低於 6GB，啟動時務必添加 `--lowvram` 參數，防止顯卡負載過重導致當機。

---

## 3. 必備管理擴充：ComfyUI Manager

為了優雅地管理複雜的插件依賴，**ComfyUI Manager** 是必裝的神裝。當您下載一個複雜的工作流 JSON 發現滿屏紅色的報錯節點時，只需點擊 **`Install Missing Custom Nodes`**，它便會自動從網路上掃描並安裝缺失的組件。

---

## 4. 進階開發：將生成引擎 API 化

這才是將 ComfyUI 整合進 Moyind 後端資產生成的關鍵：

1.  **開發者模式開關**: 在設置中開啟 **「Enable Dev mode Options」**，即可導出 **「API Format」** 的 JSON。
2.  **程式控制邏輯**: API 格式的 JSON 去除了 UI 坐標，純粹保留邏輯骨架。開發者可以透過 Python 腳本，利用 HTTP POST 將參數化的指令傳送至 `http://127.0.0.1:8188/prompt`，實現全自動、無人值守的資產渲染。

---

## 💡 技術監工的實務心得

在建構生產級的 ComfyUI 接入層時，請落實以下機制：

- **駐列 (Queue) 負載平衡**: 在高併發情境下，務必在後端實作 Task Queue，避免多個生成請求同時湧入導致顯存溢出 (OOM)。
- **WebSocket 狀態監控**: 透過 WebSocket 實時接收渲染進度。這讓系統能具備重試機制 (Retry)，確保在生成失敗時能自主修復並重新產出。

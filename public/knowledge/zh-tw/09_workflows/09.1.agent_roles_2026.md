# AI Agent 分工規範（2026 旗艦版）：Claude、Codex、Antigravity 的三位一體協作 (Agent Roles 2026)

## @概覽

到了 2026 年，將所有開發任務塞給單一 AI 已成為過時的做法。在 Moyin 專案中，我們實踐了高度專業化的「AI Agent 三鼎立體系」：由 Claude 擔任架構師、Codex 負責後端數據、Antigravity 執掌前端 UI。這種分工確保人類能穩坐「決策者」之位，而機器勞動力則被精確編排。

本規範定義了各 Agent 的職責邊界、標準產出以及協作矩陣。

---

## 1. 角色職責：立體結構化分工

### 🟢 Claude — 架構師 / 戰略規劃者

作為專案的「大腦」，負責繪製藍圖並進行高階技術決策。

- **職責**：需求分析、系統架構設計、大規模重構戰略、Code Review 頂層稽核。
- **核心優勢**：抽象推理、邏輯一致性把關、潛在風險預測。
- **標準產出**：結構化實作藍圖、稽核與重構報告、精確的 TODO 任務清單。

### 🔵 Codex — 後端 / 資料工程師

作為專案的「心臟」，掌控伺服器邏輯與數據流轉。

- **職責**：API 端點開發、資料庫設計與遷移、效能調校、非同步處理優化。
- **核心優勢**：精確代碼生成、複雜 SQL 操作、高可靠性的資料清洗。
- **標準產出**：可部署之伺服器端源碼、資料遷移腳本、單元與整合測試用例。

### 🔴 Antigravity — UI UX / 前端工程師

作為專案的「臉面」，構建視覺體驗與流暢互動。

- **職責**：視覺稿精確還原 (Figma/etc.)、前端狀態管理、共用 UI 組件封裝、快速原型開發。
- **核心優勢**：CSS/動畫細節雕琢、直覺式的組件架構設計。
- **標準產出**：精緻的前端展示層代碼、UI 組件庫、渲染驗證日誌。

## 2. 協作模式矩陣：高效開發流

### 模式 A：架構驅動（Claude 領銜 ➝ 執行層落地）

- **流程**：Claude 產出實作藍圖 ➝ Codex 或 Antigravity 汲取藍圖並鎖定邊界執行編碼。
- **適用**：從零打造的新模組、觸及根基的重構、複雜的跨系統任務。

### 模式 B：扁平執行（用戶直達指令 ➝ Codex/Antigravity）

- **流程**：用戶提供精確指令 ➝ 指定的執行層 Agent 直接修改代碼執行。
- **適用**：急件修理 (Hotfix)、常態性文字或參數維護、不涉及架構的小型任務。

### 模式 C：審核屏障（落地層完成 ➝ Claude 質檢）

- **流程**：代碼變更完成後，Claude 以客觀視角執行 Code Review ➝ 確保品質達標後方可併發。
- **適用**：正式環境部署前夕、核心功能模組之素質把關。

---

## 3. 2026 年系統級絕對底線

- **謀劃 (PLAN) 先行**：凡遇複雜任務，強制掛載 Claude 主導先行出具架構方案，切勿讓執行層 Agent 瞎猜妄動。
- **客觀證據鏈 (Evidence-Based)**：不接受未經證實之「已修復」。每一次任務完畢，必須繳納實體證據（Logs、截圖或測試輸出）。
- **人工智慧最高門禁**：若未獲人類用戶明確指令 `OK` 授權，全面封鎖任何寫入、修改或刪除代碼之行為。

這套階層化的 Agent 體系是 Moyin 對抗軟體複雜性的終極方案，讓開發效率提升至光速的同時，依然保有極高的品質。

# Moyin 精益畫布 (Lean Canvas)

## @概覽

Moyin 專案作為工業化 AI 互動引擎的核心商業邏輯，已依據精益畫布框架 (Lean Canvas) 映射於此。

---

### 1-2. 痛點與客戶細分 (Problem & Customer)

| 1. 痛點 (Problem)                                                                       | 2. 客戶細分 (Customer Segments)                                            |
| :-------------------------------------------------------------------------------------- | :------------------------------------------------------------------------- |
| **製作成本過高**：傳統動漫高達 $50k/min，獨立開發者或小團隊無法負擔。                   | **視覺小說玩家**：偏好深度劇情、角色導向敘事，且願意為優質 IP 付費的群體。 |
| **開發週期過長**：往往耗時數月甚至數年，無法隨時響應 Web3 或 TikTok 的快節奏趨勢。      | **動漫內容消費者**：短影音重度用戶，偏好碎片化且高頻更新的娛樂內容。       |
| **競品內容空洞**：市面上的 AI 生成動畫往往淪為 PPT 幻燈片或慢動作，缺乏敘事深度與情感。 | **科技愛好者/早期採用者**：對 AI 工具輔助創作者持開放與支持態度的族群。    |

---

### 3-4. 核心價值與解決方案 (UVP & Solution)

| 3. 核心價值主張 (UVP)                                                                                    | 4. 解決方案 (Solution)                                                        |
| :------------------------------------------------------------------------------------------------------- | :---------------------------------------------------------------------------- |
| **Tokyo Legal (東京法理優勢)**：利用日本《著作權法》第 30-4 條，建立一套合法訓練且合規發行的工程化框架。 | **Moyin Engine**：基於 ComfyUI 與 Rust 邏輯驅動的工業化生產流水線。           |
| **Alive Character (靈動化表演)**：透過 Director Agent 驅動的情感化演出，徹底告別傳統 AI 的僵硬感。       | **產能極大化**：目標實現週更 2 集 (共 20 分鐘)，並將成本控制在每分 $15 以下。 |

---

### 5. 渠道曝光 (Channels)

- **TikTok / Shorts**：作為頂層流量入口，負責獲客與題材驗證。
- **YouTube / Bilibili**：沉澱長度內容、建立核心粉絲社群與內容資產。
- **Steam / DLsite**：最終產品銷售、變現與 IP 價值收割的最佳出口。

---

### 6-7. 收入來源與成本結構 (Revenue & Cost)

| 6. 收入來源 (Revenue Streams)                              | 7. 成本結構 (Cost Structure)                                            |
| :--------------------------------------------------------- | :---------------------------------------------------------------------- |
| **廣告分成**：YouTube/TikTok 等平台（低毛利/高流量驅動）。 | **基礎設施**：東京電費（自有算力）、AWS/Vercel 代管、以及 AI API 令牌。 |
| **會員訂閱**：Patreon/Discord（中毛利/高黏性驅動）。       | **人力成本**：核心開發與營運時間、特定藝術資產的外包。                  |
| **產品銷售**：Steam 遊戲銷售（高毛利/擴展性強）。          | **研發預算**：自定義 LoRA 模型訓練與工作流調優。                        |

---

### 8-9. 壁壘與指標 (Moat & Metrics)

| 9. 不公平競爭力 (Unfair Advantage)                                                                             | 8. 關鍵指標 (Key Metrics)                                         |
| :------------------------------------------------------------------------------------------------------------- | :---------------------------------------------------------------- |
| **10-GPU Farm (自有算力農場)**：在東京擁有自有 GPU 集群，將試錯與實驗成本降至趨近於零。                        | **RPM (Revenue Per Minute)**：設定每分鐘產出收益目標 > $30。      |
| **Full-Stack Ownership (全棧掌控度)**：代碼 (Rust/Python) + 模型 (LoRA) + 劇本 (IP) 三位一體，極難被簡單復刻。 | **WVR (Wishlist/Visits Rate)**：Steam 商店頁面的轉化率不低於 5%。 |

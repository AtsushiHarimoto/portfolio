# 本地端小參數推理模型 (Edge AI & SLMs) 技術與商業藍圖 (SLM Edge AI Blueprint)

## @概覽

傳統 AI 多依賴雲端巨量參數模型（如 GPT-4, Claude 3），但隨著**模型蒸餾 (Knowledge Distillation)** 與**強化學習 (RLHF / GRPO)** 技術的成熟，8B 到 14B 參數級別的**小語言模型 (Small Language Models, SLMs)** 已經能在極低的硬體功耗下，匹敵過去千億參數大模型的邏輯推演與程式碼撰寫能力。

本藍圖旨在探討如何利用這些 SLMs（如 DeepSeek-R1-Distill, Qwen-2.5-Coder 等），在脫離雲端算力的前提下，將 AI 推理能力下放到「本地終端機 (Edge Device)」與用戶的個人電腦中，為 Moyin 專案開闢出無網路限制、絕對私密的全新商業獲利點。

---

## 1. 核心技術與架構關鍵

- **思維鏈推理蒸餾 (Distillation of Reasoning)**：
  過去小模型容易產生「幻覺」，但最新技術透過將超大型思維鏈模型（如 DeepSeek-R1 或 OpenAI o1）的推理過程 (CoT) 當作標籤資料，訓練出參數量小卻具備同等邏輯解題能力的模型。
- **低位元量化 (Quantization)**：
  核心突破如 AWQ、GPTQ 甚至 GGUF 格式。透過將浮點運算壓縮為 8-bit 或 4-bit 量化，即使在配備 8GB 共享記憶體的 Apple M 系列晶片，或 8GB VRAM 的家用筆電，都能流暢運行。

---

## 2. 本地化部署架構 (Local-First Architecture)

為了讓 Moyin 軟體在不同消費級硬體上「開箱即用」，技術執行必須具備高相容性：

- **本地推算引擎 (Local Engine)**：
  - **Windows / Linux**：採用 `llama.cpp` 或 `Ollama` 作為底層引擎。
  - **macOS**：對接 `MLX` 框架，最大化壓榨 Apple Unified Memory 的算力。
- **適用模型 (SLM Suite)**：
  - **通用代理控制 (Agentic Control)**：`Qwen 2.5 - 7B`，高指令遵循度與中文原生支援。
  - **程式或文本高度推理 (Deep Reasoning)**：`DeepSeek-R1-Distill-Llama-8B (GGUF)`。
  - **劇情生成與多樣化對話 (Creative Narration)**：針對 ACG 文化領域微調特化的 7B - 8B 模型。

---

## 3. 商業化路徑與應用場景

透過 SLM 模型，可完全切斷對外部 API（如 OpenAI / Anthropic）的依賴，徹底扭轉按次付費的成本結構。

### 3.1 創作者端：無限制的離線創作終端 (Limitless Creator Desktop)

- **痛點**：對於需要大量產出腳本、社群貼文的使用者而言，每月的 API Token 費用是巨大的隱形成本壓力。
- **解決方案**：
  - 打包成桌面可執行檔（Electron 或 Tauri 架構）。
  - 使用者安裝後，軟體後台自動運行輕量型的 `Llama.cpp` 引擎接管本地顯卡。
- **變現邏輯**：採用「軟體本體買斷 (Lifetime License) + 官方精選微調模型包 DLC」形式。讓用戶感覺「買了就能無限制、無限量地產出內容」。

### 3.2 遊戲端：本地算力驅動的 NPC 互動引擎 (Local AI-Driven NPC)

- **痛點**：雲端驅動的 NPC 易遭遇延遲（破壞沈浸感）、斷網即崩潰，且營運方面臨持續的 API 開銷。
- **解決方案**：
  - 遊戲啟動時連帶拉起專為低階硬體量身定製的 1.5B ~ 3B 級別小模型 (如 `Qwen-2.5-1.5B`)。
  - 僅佔用少於 3GB 的 VRAM 記憶體。
- **變現邏輯**：作為遊戲本身的**獨特賣點 (USP)**：宣傳為「會利用您的顯示卡，讓世界鮮活起來的 AI 單機遊戲」，且保證「永遠不關服、零伺服器依賴」。

---

## 4. 落地實作第一步 (Next Action Items)

1. **框架整合**：研究將 `Ollama` 或 `llama.cpp` 編譯進 Tauri / Electron 軟體的自動化打包方案。
2. **效能及顯存測試**：在目標硬體上跑測試腳本，衡量 Tokens Per Second (TPS) 與顯存佔用是否合乎流暢對答的底線 (> 15 TPS)。
3. **混合架構兼容**：在 `moyin-web` 的設定頁面新增一個介面，讓使用者可以勾選「使用雲端 AI API」還是「啟用本機 AI 加速模式」。

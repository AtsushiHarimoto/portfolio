## 1. 離線模型中樞：Ollama 部署實戰

如果你家裡有一張還不錯的顯示卡 (NVIDIA RTX 系列)，那我就求求你別再去擠 OpenAI 的獨木橋了。去裝個 **Ollama** 吧，這玩意能讓你在本機一鍵啟動各種神級開源模型。

### 📦 依照你的「顯存 (VRAM)」來挑老婆 (模型選型)

你的顯卡有幾 GB 記憶體，直接決定了你能跑多聰明的 AI：

- **2-4GB (文書機)**：只能跑 `llama3.2:3b`。雖然笨了一點，但拿來測格式或是跑簡單的 NPC 對談勉強夠用。
- **6-8GB (主流規格 / RTX 3060等)**：**黃金標配！** 請無腦選擇 `qwen2.5:7b`。阿里的通義千問在中文與日文語感上真的極強，是目前 Galgame 引擎的首選。
- **12GB (高階卡)**：可以上 `qwen2.5:14b`。推理深度明顯上了一個檔次。
- **24GB (3090/4090 課長)**：恭喜你，你可以直接體驗 `32b` 甚至 `qwq` 預覽版模型的恐怖智商，記憶力與邏輯強到沒朋友。

## 2. 雲端認證黑魔法：Firefox Cookie 劫持方案

有時候我們必須用到雲端最強模型（比如 Gemini 1.5 Pro），但官方 API 限制很多，或者你想用網頁版的免費額度。問題來了：現在各大平台（Google, Claude）的 Cookie 加密得像鐵桶一樣，普通的 Python 腳本根本偷不到。

### 🦊 為什麼我們選用 Firefox？

因為 Chrome 把 Cookie 鎖死在電腦的系統硬體密鑰裡，解密起來會瘋狂觸發防毒軟體警報。但 **Firefox** 比較講義氣。它的 Cookie 存在一個相對單純的 SQLite 資料庫裡。

**Moyin 的工作流是這樣的**：

1. 啟動腳本時，自動幫你開啟 Firefox。
2. 你手動在網頁版登入好。
3. 我們的 `cookie_check.py` 腳本會在背景直接「瞬移」進 Firefox 的本地儲存區，把認證 Token 完好無損地提取出來給 API Gateway 使用。
   這種做法成功率近乎 100%，而且穩定得不像話。

## 3. 毫秒級響應：Native Context 續聊技術

如果你在用本機 Ollama，別再去用外面那些「包裝好」的相容介面了。為了追求視覺小說那種「秒點秒出」的快感，我們直接對接了 Ollama 原生的 `/api/chat` 協定。

- **KV Cache 複用**：當上一句對話結束，Ollama 會回傳一個叫 `context` 的記憶體快照。
- **降維打擊**：下一句對話時，我們不再把之前的歷史重傳一次，我們只給它這串「數字快照」加上最新的一句話。
  這會讓原本要跑 3 秒的推論，縮短到不到 0.5 秒。這才是專業工程師該有的追求。

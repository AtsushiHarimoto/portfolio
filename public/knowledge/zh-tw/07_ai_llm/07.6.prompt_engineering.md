# 支配 AI 的權能：提示詞工程學 (Prompt Engineering) (Prompt Engineering)

## @概覽

大家好，我是 AKIRA。
這期我們來聊聊這兩年被吹上天的：**提示詞工程 (Prompt Engineering)**。
很多人以為這門學問是在玩「文學創作」，在那邊絞盡腦汁寫一些感性抒情的小說大綱。
**我告訴你，如果你想開發真正的 AI 軟體，提示詞不是文學，它是「工程控制」！**

AI (大語言模型) 本質上就像是一個演技極佳、但完全沒有常識的「即興劇演員」。如果你沒給它精確的劇本規範，它產出的內容會發散到讓你崩潰，甚至把你的後端資料夾全部炸掉。
今天我們就來拆解，Moyin 專案是怎麼用「絕對控制力」把這群野馬般的 AI 給馴服成乖寶寶的。

---

## 1. 提示詞雙軌制：誰才是真正的導演？

在我們的 API 協議裡，我們把指令切成兩種完全不同的維度，絕對不能混為一談：

### 👑 System Prompt (全域系統指令)

這就是「導演的隱形擴音器」。它深埋在程式後端，普通玩家跟使用者根本看不到。

- **權威性**：它的權重最高，不管玩家在外面怎麼威脅利誘，AI 內心深處死都要守住這條底線。
- **任務**：錨定 NPC 的人格（比如：「你現在是高冷的林晚，只能用全形標點」、「強制回傳 JSON 格式，不准說廢話」）。

### 🗣️ User Prompt (場景與玩家輸入)

這就是「舞台上發生的即時台詞」。

- **任務**：傳達瞬息萬變的互動狀態。例如：「玩家摸了摸你的頭」，或是系統塞進去的環境參數：「【當前環境】晚上十點，月黑風高」。

---

## 2. 輸出格式的「死命令」：JSON 強制約束

這是我最常強調的一點。
對於後端程式來說，AI 寫出來的劇本再美都沒用，如果它在回傳結果前多說了一句：「好的，親愛的，這是你要的劇情：...」。
**恭喜你，你的後端 JSON 解析器會當場崩潰報錯！**

### 🛡️ 實戰防禦三部曲：

1.  **一擊必殺範本 (One-Shot Example)**：別只會用講的！直接在 System Prompt 底下塞一個「完美的輸出 JSON 格式」給它看。跟它說：「不准多一個字、不准少一個括號」。
2.  **思維鏈隱藏術 (Hidden CoT)**：如果你需要 AI 思考（比如先判斷有沒有中計，再決定回話），我們可以請它在 `<thought>` 標籤裡思考。但記得，我們在 API 閘道端會用正則表達式把這整段「思考雜質」全部濾掉，只把最純的 JSON 傳給前端。
3.  **自癒重試機制 (Self-Healing)**：萬一 AI 真的斷路、括號沒寫完怎麼辦？我們的 GateWay 會捕獲這個異常，然後把錯誤訊息丟回去給 AI 說：「你剛才吐的大便是壞的！第 4 行少了個逗號，現在、立刻、給我修好重傳！」

---

## 3. 踩過才知道的：格式崩潰雷區 💣

在和程式對接時，這兩個雷區你一定要死命防守：

- **Markdown 字首污染**：AI 很愛在 JSON 前後加 \`\`\`json 標籤。這在網頁上看很美，在程式裡就是死罪。請在 Prompt 裡反覆恐嚇它不准加，或者在後台程式碼裡強行 `replace` 掉。
- **雙引號地獄 (Quote Hell)**：JSON 的語法是用英文雙引號包裹的。如果你的對話劇本也是用 `"吃了嗎？"`。兩邊的引號一撞車，JSON 解析就直接裂開。
  - **解決方案**：強制命令 AI 在寫對話時，**一律改用全形中文引號「 」**！這能保證技術層面與內容層面完美視覺隔絕。

## 💡 Vibecoding 工地監工發包訣竅

在要求 AI 幫你寫這種高強度限制的 Prompt 模板時，請拿出你身為大魔王的氣勢：

> 🗣️ `「AI 助教！給我聽好！我現在要設計這套 NPC 核心對話接口的 System Prompt！
第一，我要求你採用【強迫 JSON 輸出模式】，如果你的回應裡夾帶了半句解釋性的廢話，我會判定解析失敗並重啟你！
第二，給我導入【One-Shot 範例引導】。把你預期的 narrative 內容、state_delta、以及 NPC 心境，全部給我格式化好寫進 Template 裡。
第三，為了防止 JSON 結構混亂，立刻給我聲明：對話內容必須採取全形【「 」】符號進行包裹！
最後，給我設計出一套 Retry 容錯機制，如果解析崩潰，自動提示錯誤點並進行二次生成！現在給我產出這套穩定度 100% 的提示詞架構！」`

記住，好的提示詞工程，是讓 AI 覺得它根本沒得選擇，只能乖乖聽命於你。

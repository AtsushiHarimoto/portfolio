# 燒錢還是賺錢？AI Galgame 引擎的 API 續命與變現戰略 (API Strategy)

## @概覽

大家好，我是 AKIRA。
這期我們來講講一個非常現實、甚至會決定你這間獨立遊戲工作室能不能活過下個月的殘酷問題：**API 的帳單與商業變現**。

現在大家都在瘋 AI 驅動的互動遊戲、AI 女友，或是解謎 Galgame。聽起來很美，但如果你只會在那邊無腦串接 OpenAI，我保證你遊戲還沒上線，你的信用卡額度就會被那幾百萬個 Token 給直接刷爆！

這期我們就來聊聊，Moyin 專案是怎麼在追求極致智力的同時，把 API 成本死死地按在地上摩擦的。

---

## 1. 錢是怎麼噴掉的？Token 爆炸的兩大死因

在做互動敘事遊戲時，新手最常死在這兩種情況：

1.  **💸 無狀態協議的智商稅**：官方 API (如 GPT-4o) 其實是個失憶症患者，它完全不記得你剛才說了什麼。所以為了讓 NPC 記住她是你的青梅竹馬，你每次按下對話，都得把前面的幾千字、甚至幾萬字的世界觀跟歷史對話「全部重新傳一次」！隨這遊戲越玩越久，那個 Input Token 的增長速度簡直就像火箭升空。
2.  **🚨 逆向 API 的奪命追魂扣**：很多人為了省錢去搞那些偷偷破譯 (Reverse) 網頁版的免費 API。我跟你講，那是拿命在玩！那些東西極度不穩定，今天能跑，明天平台一升級防護，你的遊戲就當場癱瘓噴出 `403 Forbidden`。如果你是想正式營運賺錢，這種隨時會斷線的黑科技絕對不能碰。

---

## 2. 只有強者才懂的：三階段引擎演進法

為了在「智力」、「延遲」與「錢包」這三個不可能三角形中找到生存之道，我們 Moyin 引擎採取了這三階防禦與進化路線：

### Phase 0: 狀態機與殘酷的滾動摘要 (Rolling Summary)

這一階是保命基礎。我們不再把歷史對話全部塞給 AI，我們在引擎裡維護一個精密的 `state.json`，只把當下最重要的變數丟給 AI。更絕的是，我們實行「滾動淘汰制」：AI 只看最近 8 輪對話，超過的部分就立刻丟進壓縮機變成一句話的「回憶摘要」。這樣一來，不管玩家玩到天長地久，我們每回合的 Token 成本永遠都被鎖死在極低的常數範圍內，徹底根除成本爆炸。

### Phase 1: 前後大腦拆分：控制層與寫手層之解耦

這是我最得意的設計。我們放兩個 AI！門口站一個最便宜、最爛的「小腦 (Controller)」負責過濾玩家的廢話。只有當小腦判定玩家這句話真的觸發了關鍵劇情、或是精確的解謎時，我們才喚醒背後那個極度聰明但超級昂貴的「大腦 (Writer - 比如 Claude 3.5)」出來寫一段神級劇本。這叫「漏斗攔截法」，能節省掉 80% 以上不必要的頂級模型調度費。

### Phase 2: 終極型態：全域在地化推理 (Fully Localized)

這才是我們的終局戰場：**不聯網，直接在玩家的電腦上跑模型！** 透過 `llama.cpp` 跟量化黑魔法，我們現在能把 8B 級別的高性能模型 (如 Llama-3 或 Qwen) 直接塞進玩家那張 RTX 30/40 的顯卡裡。

- **好處一**：API 成本變成「零」。不用再付錢給 OpenAI 買那些貴得要死的點數。
- **好處二**：隱私無敵。玩家跟女主角說的那些不能說的祕密，死也傳不回伺服器。
- **好處三**：秒吐字。沒有網路延遲，體驗直接起飛。

## 3. 怎麼從這堆 AI 程式碼裡挖到金礦？ (Commercialization)

1.  **初期：自帶金鑰 (BYOK - Bring Your Own Key)**：這是目前最健康的單機遊戲模式。玩家去申請自己的 API Key 填進來。我們賣的是什麼？**我們賣的是這套精湛的「Prompt 框架」跟「記憶管理演算法」！** 底層智力是公共財，我們的「靈魂結構」才是價值所在。
2.  **中期：無感借用 (Cookie Capture)**：讓玩家自己登入他們的 Claude 或 GPT 網頁版，把權限借給遊戲。這能讓我們在遊戲開發初期「完全沒有流量負擔」，狂蹭官方的免費額度，專心拓荒社群。
3.  **後期：軟硬體授權**：針對那種家裡有頂規硬體的大戶，我們銷售「本地化模型專屬包」，讓他們享受極致的原生推理體驗。

## 💡 Vibecoding 工地監工發包訣竅

在使喚 AI 幫你設計這種帶有商業變現能力的 LLM 接入層時，請拿出你身為產品經理的狠勁，對著 AI 這樣詠唱：

> 🗣️ `「AI 助教！給我聽好！我現在要設計這款 Galgame 的 LLM 接入中控台。
我絕對不允許你寫出那种【一股腦把對話全傳過去】的無腦程式碼！我要求你立刻導入【v0 滾動摘要機制】！
你必須在後台維護一個 state_delta 的 JSON 協議，所有長度超過 10 輪的對話紀錄，通通給我強制壓縮成 200 Tokens 以內的【歷史回憶錄】！
同時，你在設計 API 接口時，給我預留好【Provider 策略模式】，未來我要能一鍵把流量切換到本地的 Llama.cpp 端點，不要讓我看到你的架構代碼寫死了！立刻去建構這套節流引擎！」`

不要讓 Token 的火燒掉你的創業夢，精打細算的架構師，才能活到最後。

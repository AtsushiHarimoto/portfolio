# Claude Reflect 自動規則沉澱系統 (Claude Reflect)

## @概覽

AI Agent 就像剛入職的菜鳥警員，容易重複犯同樣的錯誤，你糾正後下次又走回老路。「只靠口頭警告讓模型記住」是極度不可靠的，我們需要「白紙黑字讓專案 (repo) 記住」。

`claude-reflect` 就是一個能把你的「長官訓話」變成「可強制執行的警察通例 (版本化規則文件)」的神兵利器！

---

## 1. 核心理念：把口頭糾錯變成「程式碼憲法」

- **核心理念**：把「口頭糾錯」變成「可強制執行的 ESLint 般規則」，直接寫入 `CLAUDE.md` / `AGENTS.md` / `Skills`。
- **生活比喻**：就像是警隊的內部調查科 (C&I)，負責收集長官對警員的投訴，然後每晚自動編成《警務守則第一條》，明天全警局強制遵守。
- **工作原理**：捕獲 (記錄投訴) → 隊列 (收集案件) → 提煉 (總結守則) → 人工 Review (長官最後簽名) → 寫入 → Git 版本化。

---

## 2. 核心指令速查

### 日常沉澱（每日下班結案）

| 指令                 | 用途                     | 前端 / 比喻                                   |
| -------------------- | ------------------------ | --------------------------------------------- |
| `/reflect --review`  | 查看待處理的學習信號     | 看今天警員收到了多少市民與長官的投訴單        |
| `/reflect --dry-run` | 預覽將改什麼，不實際寫入 | 虛擬執行 (看報告草稿，不正式發出 ESLint 規則) |
| `/reflect`           | 正式處理並寫入規則       | 長官正式簽名蓋章，將規則寫入《警務通例》      |

### 歷史掃描與優化

| 指令                      | 用途                       |
| ------------------------- | -------------------------- |
| `/reflect --scan-history` | 掃描歷史 session，補撈規則 |
| `/reflect --dedupe`       | 語意去重，合併相似規則     |
| `/reflect-skills`         | 從近 14 天發現重複模式     |

---

## 3. 沉澱原則：什麼值得成為《警務通例》？

### ✅ 值得沉澱：

- 糾正過 2 次以上的重複錯誤 (屢教不改的坑)。
- 反覆浪費 debug 時間的問題。
- 能寫成一句**強烈且可執行**規範的 (例如：永遠不要用 `var`)。

### ❌ 不值得沉澱：

- 只有一次性的特殊 bug。
- 這週暫定方案，下週可能又會推翻的邏輯。
- 非常主觀、無法透過程式碼驗證的規則。

---

## 4. Moyin 專案整合現狀

1. **每日下班前的儀式**：執行 `/reflect --review` 確認投訴單 → `/reflect` 結案。
2. **每週整理**：執行 `/reflect --dedupe` 清理過期或重複的警司命令。
3. **智慧共享**：將 `CLAUDE.md` 的變更納入 Git commit，與所有人和 AI 共用這份產出的智慧。

掌握了 Claude Reflect，就代表你的 AI 助手不再只是個「聊天機器人」，而是個有紀律、且會跟著團隊一同成長的「Agentic 引擎」。
